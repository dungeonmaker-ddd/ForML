Day 01 · Page 1

ML 本质 — y = wx + b 是一切的起点

机器学习（Machine Learning）的核心：从数据中自动学习规律，产生模型，用模型预测。与传统编程的本质区别 ——

传统编程
输入：规则 + 数据
输出：结果
人写 if-else，数据仅用于查询
机器学习
输入：数据 + 结果
输出：规则（模型）
机器自动从数据中提取规律

AI ⊃ ML ⊃ DL：三者是严格包含关系，不是并列。AI 最大（含专家系统、模糊逻辑等），ML 是 AI 的核心子集，DL（深度学习）是 ML 中基于多层神经网络的方法子集。

ŷ = wx + b
最简线性模型 — 理解 ML 的起点
w
斜率 / 权重
衡量特征对标签的影响力
模型参数：从数据中学来
b
截距 / 偏置
基础偏移量
模型参数：算法自动调整
Key
参数 vs 超参数：w、b 是模型参数（从数据学来）；K 值（KNN 中）是超参数（人为设定，不从数据学来）。
Glossary
ML 机器学习从数据自动学习规律并产生模型
DL 深度学习ML 子集，基于多层神经网络
Parameter 参数模型内部变量，如 w、b，由算法从数据中学习
Hyperparameter模型外部设定值，如 K、学习率，需人为指定

数据结构 — X 矩阵 · y 向量 · 划分铁律

ML 的数据集本质上是一张二维表格（矩阵）：每行是一个样本，每列是一个特征，末列通常是标签。

X ∈ ℝm×n    y ∈ ℝm
m 个样本 × n 个特征 = 特征矩阵 X  |  m 个标签值 = 标签向量 y
特征 x₁特征 x₂…标签 y
样本 1面积楼层…售价
样本 2面积楼层…售价
……………
Iron Rule
数据划分铁律：先 split → 再 train → 最后 evaluate。训练集（80%）学参数，测试集（20%）评泛化。绝对禁止用训练数据做测试 — 否则数据泄露（data leakage），评估虚高。
from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.2, random_state=22 ) # 返回四元组：训练特征、测试特征、训练标签、测试标签
Glossary
Sample 样本数据集中的一行，一个观测单元
Feature 特征数据集中的一列，描述样本的属性
Label 标签模型要预测的目标列
Data Leakage测试集信息渗入训练过程，导致评估失真

监督学习 — f(X) → Y · Loss · 回归 vs 分类

监督学习：已知 X（特征）和 Y（标签），学习映射函数 f 使 f(X) ≈ Y。训练过程：预测 ŷ = f(X) → 计算 loss → 调整参数 → 重复直到收敛。

MSE = 1/m · ∑(yi − ŷi)²
均方误差 — 回归任务最常用的损失函数，衡量预测值与真实值的差距
回归 Regression
输出：连续值
案例：房价预测 ŷ = 358 万
评估：MSE · MAE · R²
判断：结果能做加减法 → 回归
分类 Classification
输出：离散类别
案例：猫狗分类 ŷ ∈ {猫, 狗}
评估：准确率 · F1
判断：结果不能做加减法 → 分类
Key
面试追问：房价预测是回归还是分类？→ 回归，因为输出是连续数值。垃圾邮件检测？→ 分类，输出 0 或 1。
Glossary
Supervised 监督学习有标签的学习方式，学习 f(X)→Y 映射
Loss 损失函数衡量预测值与真实值的差距
Regression 回归输出连续值的监督学习任务
Classification 分类输出离散类别的监督学习任务

无监督 · 半监督 · 强化 · 四种学习方式

学习方式输入目标反馈
监督X + Y学 f(X)≈Y有 (loss)
无监督仅 X发现 P(X) 结构无
半监督少量Y + 大量X降低标注成本部分
强化状态 + 奖励max ∑γtRt延迟奖励

无监督学习：只有特征 X，没有标签 Y。目标是发现数据的内部结构和分布规律。典型方法：聚类（K-Means、DBSCAN）— 按相似性自动分组，物以类聚。

强化学习四要素：Agent（智能体）、Environment（环境）、Action（行动）、Reward（奖励）。通过试错找到累计奖励最大的策略。案例：AlphaGo、机器人控制。

Q2 面试
监督 vs 无监督的核心区别？
一句话：有无标签是根本区别。监督有标签、有反馈、预测已知类型；无监督无标签、无反馈、探索未知结构。选择依据：数据形态决定学习方式。
Glossary
Unsupervised无标签学习，发现数据内部结构
Clustering 聚类按相似性将样本自动分组
Semi-supervised少量标签 + 大量无标签数据的学习
Reinforcement通过试错和奖励信号学习最优策略

五步 Pipeline — sklearn 建模模板

1
获取数据搜集任务相关数据集（公开数据集 / 爬虫 / 传感器）
2
数据处理缺失值填充 / 异常值检测 / 去重 → 准备干净数据
3
特征工程提取 · 预处理 · 降维 · 选择 · 组合 — 占 60%–80% 时间
4
模型训练选算法（线性回归 / 决策树 / KNN / SVM）→ fit(X, y)
5
模型评估score(X_test, y_test) → 效果好上线，差则回退迭代
from sklearn.datasets import load_iris from sklearn.model_selection import train_test_split from sklearn.preprocessing import StandardScaler from sklearn.neighbors import KNeighborsClassifier X, y = load_iris(return_X_y=True) X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2) X_train = StandardScaler().fit_transform(X_train) clf = KNeighborsClassifier(n_neighbors=5).fit(X_train, y_train) print(clf.score(X_test, y_test))
核心洞察
数据和特征决定 ML 上限，模型和算法只是逼近这个上限。再好的算法也救不了烂数据。新手误区：花大量时间调参，忽视数据和特征质量。
Glossary
Pipeline 流水线ML 建模的标准化多步流程
fit() 训练模型从数据中学习参数
score() 评估在测试集上计算模型准确率
Iteration 迭代评估不佳时回退调整再重新训练

特征工程 — 五大子领域

特征工程：利用专业知识处理数据，让模型更容易学到规律。包含五大子领域：

子领域做什么举例
提取非结构化 → 数值矩阵图像 28×28 → 784 维向量
预处理消除量纲差异归一化 / 标准化
降维高维 → 低维PCA 主成分分析
选择挑选相关特征子集过滤法 / L1 正则化
组合创造新特征BMI = 体重/身高²
BMI = weightkg / heightm²
特征组合示例 — 领域知识驱动，有医学定义，高可解释性
Key
鸢尾花启示：为何不用颜色作特征？专业人员判断颜色对三种鸢尾花区分力不够。特征不是越多越好 — 无关特征是噪声，可能导致维度灾难和过拟合。
Glossary
Feature Engineering利用专业知识处理数据的技术总称
Extraction 提取从原始数据提取数值特征
Feature Crosses用数学运算组合多个特征为新特征
Dimensionality特征数量，即数据的维度

归一化 — MinMaxScaler 公式与 API

x' = (x − xmin) / (xmax − xmin)
将特征映射到 [0, 1] 区间 — 所有特征压缩到同一尺度

计算示例：数据 [10, 20, 30, 40, 50]，min=10，max=50。
x'(30) = (30−10)/(50−10) = 0.5  |  x'(10) = 0  |  x'(50) = 1

from sklearn.preprocessing import MinMaxScaler scaler = MinMaxScaler(feature_range=(0, 1)) X_train_scaled = scaler.fit_transform(X_train) # 学习 min/max 并转换 X_test_scaled = scaler.transform(X_test) # 只转换，不学习 # 属性：scaler.data_min_, scaler.data_max_
Warning
致命缺陷：如果 max/min 是异常值，全部数据被过度压缩。例：正常 [10, 20, 30] + 异常值 1000 → 归一化后正常数据全挤在 [0, 0.03]，区分度丧失。仅适合：已知范围、无异常值、图像像素 0–255。
Glossary
MinMaxScaler将特征缩放到指定范围的归一化工具
fit_transform()学习参数并执行转换（仅用于训练集）
transform()用已学参数执行转换（用于测试集）
Normalization将数据缩放到固定范围的预处理方法

标准化 — StandardScaler · N(0, 1)

x' = (x − μ) / σ
转换为标准正态分布 N(0, 1) — 均值 = 0，标准差 = 1
from sklearn.preprocessing import StandardScaler scaler = StandardScaler() X_train_scaled = scaler.fit_transform(X_train) X_test_scaled = scaler.transform(X_test) # 属性：scaler.mean_ (均值), scaler.var_ (方差)

正态分布 N(μ, σ²)：μ 决定中心位置，σ 决定分散幅度。标准化后 N(0,1)：1σ 覆盖 68%，2σ 覆盖 95%，3σ 覆盖 99.7%。

归一化 MinMaxScaler
受极值 max/min 支配
鲁棒性差
适合：小数据、已知范围
标准化 StandardScaler
受均值 μ 和 σ 驱动
鲁棒性强
适合：大数据、有噪声
Key
面试必答：归一化和标准化的本质区别？→ 归一化用极值，标准化用统计量（均值/标准差），鲁棒性不同。实践结论：不确定时优先选标准化。
Glossary
StandardScaler将特征转换为均值 0、标准差 1 的标准化工具
μ (mu) 均值所有数据的算术平均
σ (sigma) 标准差衡量数据离散程度，σ² 为方差
N(0,1)标准正态分布，均值为 0、标准差为 1

PCA 降维 — 线性组合 · 方差最大化

PCA（主成分分析）：将 n 维数据投影到 k 维（k < n），保留方差最大的方向作为新坐标轴。新特征 = 原始特征的线性组合。

PC1 = w11x1 + w12x2 + … + w1nxn
第一主成分 — 方差最大方向  |  PC2 ⊥ PC1：与第一主成分正交

示例：身高 x₁ + 体重 x₂ → PC₁ = 0.7·x₁' + 0.7·x₂'（标准化后），含义 ≈ "体型大小"，无明确单位。2 维 → 1 维，保留主要方差信息。

from sklearn.decomposition import PCA pca = PCA(n_components=2) # 降到 2 维 X_reduced = pca.fit_transform(X) print(pca.explained_variance_ratio_) # 各主成分方差解释比 # 选 k：累计方差解释比 ≥ 95%
PCA 降维
纯数学优化，无语义
不可解释
仅限线性组合
自动化
特征组合
领域知识驱动，有语义
高可解释 (如 BMI)
任意函数形式
人工设计
Glossary
PCAPrincipal Component Analysis，主成分分析
Variance 方差数据的离散程度，PCA 保留方差最大方向
explained_variance_ratio_各主成分解释的方差占比
n_components指定降维后保留的维度数

拟合问题 — 多项式复杂度 · 正则化

模型复杂度决定拟合状态 — 不能太简单（欠拟合），也不能太复杂（过拟合）：

模型参数数状态表现
y = kx + b2欠拟合train ↑ test ↑
y = ax²+bx+c3正好拟合train ↓ test ↓
高次多项式n过拟合train ↓↓ test ↑
欠拟合 Underfitting
训练差、测试也差
原因：模型太简单
对策：增加复杂度 / 增加特征
过拟合 Overfitting
训练好、测试差
原因：模型太复杂 / 数据脏
对策：正则化 / 早停 / 数据增强
L1: Loss + λ∑|wi|     L2: Loss + λ∑wi²
L1 产生稀疏解（部分 w→0，特征选择）  |  L2 产生平滑解（所有 w 趋近 0） |  λ 控制约束强度
Glossary
Underfitting模型太简单，连训练数据规律都没学到
Overfitting模型太复杂，记住了噪声而非规律
Regularization通过约束参数大小来防止过拟合
λ正则化强度，越大约束越强

Loss 曲线诊断 — 泛化 · 奥卡姆剃刀

横轴：训练轮次（epoch），纵轴：loss 值。通过 train-loss 和 test-loss 两条曲线诊断拟合状态：

— 实线 = train-loss    - - 虚线 = test-loss    过拟合时用 Early Stopping：连续若干轮 test-loss 不下降 → 停止训练。

Key
泛化（Generalization）：模型在未见数据上的表现。ML 的终极目标不是记住训练集，而是学会规律。
奥卡姆剃刀：泛化误差相同时选更简单的模型 — 简单模型不易过拟合。
面试追问：为什么不直接用最复杂模型？→ 过拟合风险高 + 训练成本高 + 解释性差。
Glossary
Generalization模型在新数据上的泛化表现
Early Stopping验证 loss 上升时提前终止训练
Occam's Razor同等效果下优先选择更简单的模型
Epoch模型遍历一次完整训练集的过程

KNN 算法 — 5 步流程 · 分类 vs 回归

K-Nearest Neighbors：找特征空间中 K 个最近邻，用它们的类别/值决定未知样本。惰性学习：无训练过程，存储全部数据，预测时 O(m·n) 遍历。

1
计算距离d(xq, xi) 对所有训练样本 i
2
距离排序按 d 升序排列
3
取前 K 个最近邻 {xnn1, …, xnnK}
4
决策分类：ŷ = mode(投票)  |  回归：ŷ = 1/K · ∑ynni(均值)
5
输出分类：离散标签  |  回归：连续数值
Key
分类 vs 回归的唯一区别在第 4 步：分类用投票（多数表决），回归用均值。前 3 步完全相同。
面试追问：KNN 有训练过程吗？→ 没有，它是惰性学习（lazy learning）。
Glossary
KNNK-Nearest Neighbors，K 近邻算法
Lazy Learning不构建参数模型，预测时才计算
mode()众数，出现次数最多的值
mean()均值，所有值的算术平均

距离度量 — 欧氏 · 曼哈顿 · 闵氏统一框架

距离公式直觉
欧氏√∑(xi−yi)²直线距离（勾股定理）
曼哈顿∑|xi−yi|街区距离（横平竖直）
切比雪夫max|xi−yi|棋盘距离（国王一步）
d = (∑|xi − yi|p)1/p
闵可夫斯基距离 — 统一框架：p=1 曼哈顿  |  p=2 欧氏  |  p→∞ 切比雪夫

p → ∞ 推导：最大分量 |xi−yi|p 远超其他分量之和，开 p 次根后其他项贡献 → 0，极限只剩 max|xi−yi| = 切比雪夫距离。

Warning
量纲警示：特征量纲不同 → 大量纲特征主导距离。例：年收入（万级）vs 年龄（十级）→ 距离几乎只由收入决定。这就是为什么 KNN 必须做特征预处理！
Glossary
Euclidean欧氏距离，两点间直线距离，KNN 默认
Manhattan曼哈顿距离，各维度差的绝对值之和
Minkowski闵氏距离，参数 p 统一三种距离
Chebyshev切比雪夫距离，各维度差的最大值

K 值选择 + KNN API

K 是 KNN 的唯一核心超参数。K 值反向控制模型复杂度：K 越小越复杂，K 越大越简单。

K 过小 (如 K=1)
极小邻域 → 噪声敏感
决策边界极复杂
本质：过拟合
K 过大 (如 K=N)
看全部样本 → 总预测多数类
丧失局部判别力
本质：欠拟合
from sklearn.neighbors import KNeighborsClassifier # 分类器 clf = KNeighborsClassifier(n_neighbors=5) clf.fit(X_train, y_train) y_pred = clf.predict(X_test) # 输入必须二维！ acc = clf.score(X_test, y_test) # 准确率 # ⚠️ predict 陷阱 clf.predict([[1]]) # ✅ 正确：二维矩阵 clf.predict([1]) # ❌ 报错：sklearn 要求 (m,n) 形状
Key
实践经验：取较小奇数（3/5/7/9）避免平局，用交叉验证遍历候选 K 值选最优。K=5 通常是不错的默认值。
面试追问：predict([1]) 为什么报错？→ sklearn 统一要求矩阵输入，即使单样本也要双层方括号。
Glossary
n_neighborsKNN 的 K 值参数，默认 5
predict()输入特征矩阵，输出预测标签
Cross Validation交叉验证，多次划分取平均评估
Decision Boundary决策边界，分隔不同类别的曲面
