> 来 学习率怎么选 是否是和 评估强相关？                                         

● 对，学习率和评估直接挂钩——你判断学习率好不好，唯一的依据就是看 loss
  曲线怎么变。

  学习率怎么选

  核心判断方式：画 loss 随迭代次数的变化曲线

  α = 0.001（太小）          α = 0.01（合适）           α = 1.0（太大）

  loss                       loss                       loss
   |████                      |████                      |█  █   █
   |████                      |██                        |█ █ █ █ █
   |████                      |█                         |█  █   █  █
   |███                       | █                        |        █  █
   |███                       |  ·                       |          █
   |███                       |   ··                     |
   |██                        |    ···                   |    💥 发散
   +──────── 迭代次数         +──────── 迭代次数         +──────── 迭代次数
    还在慢慢降                 快速降到底                  来回震荡或爆炸
    1000次还没收敛             200次就收敛了

  判断规则就三条：

  ┌────────────────────────┬────────────┬───────────┐
  │     loss 曲线表现      │    诊断    │   对策    │
  ├────────────────────────┼────────────┼───────────┤
  │ 稳步下降，最终走平     │ 学习率合适 │ 不用动    │
  ├────────────────────────┼────────────┼───────────┤
  │ 下降极慢，1000次还在降 │ 学习率太小 │ 乘以 3~10 │
  ├────────────────────────┼────────────┼───────────┤
  │ 震荡、上升、或爆炸     │ 学习率太大 │ 除以 3~10 │
  └────────────────────────┴────────────┴───────────┘

  实际调参流程

  不需要猜，按数量级试：

  尝试顺序：α = 0.001 → 0.003 → 0.01 → 0.03 → 0.1 → 0.3 → 1.0
                                                           ↑
                                                      每次乘大约3倍

  每个跑几百次迭代，看 loss 曲线：

  for lr in [0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1.0]:
      w, b, losses = gradient_descent(X, y, lr=lr, epochs=500)
      plt.plot(losses, label=f'α={lr}')

  画出来一眼就能看出哪个好——下降最快且不震荡的那个就是最佳学习率。

  和评估的关系

  你说得对，学习率和评估强相关，但要分清两层：

  第一层：训练过程中的评估（选学习率用的）
  - 看的是 loss 曲线（训练集上的 MSE 随迭代次数变化）
  - 目的：确保梯度下降正常收敛
  - 这不是最终评估，只是调参工具

  第二层：训练完成后的评估（评价模型好坏）
  - 看的是 测试集上的 R²、MSE
  - 学习率选对了 → 模型收敛到最优 → 测试集指标才有意义
  - 学习率选错 → 模型没收敛 → 测试集指标必然差

  关系链：

  学习率合适 → loss正常下降 → w和b走到最优 → 测试集R²高
  学习率太大 → loss震荡     → w和b没走对   →
  测试集R²低（不是模型不行，是没训好）
  学习率太小 → loss没降完   → w和b停半路   → 测试集R²低（也不是模型不行）

  所以学习率不影响模型能力的上限，只影响你能不能达到那个上限。线性回归用正规方程
  能到 R²=0.85，那梯度下降调好学习率也能到
  0.85。调不好就到不了，但这是你的问题不是模型的问题。

  实际工作中

  sklearn 的 SGDRegressor 默认学习率策略是 eta0=0.01 +
  自动衰减（随迭代次数逐渐减小），大多数情况不用手调。深度学习里常用 Adam
  优化器，它自动调整每个参数的学习率，也不太需要手选。

  所以学习率调参这件事，你理解原理就够了，实际工作中框架帮你做了大部分。

> 那么 评估的方式有几种？ SGDRegressor 是什么                                   

● 评估方式

  梯度下降过程中用到的评估，其实你 day03 学的那几个指标都能用，但场景不同：

  训练过程中的评估（调学习率）

  只看一个东西：loss 曲线（MSE 随迭代次数的变化）

  epoch 1:   MSE = 45.2
  epoch 10:  MSE = 12.8
  epoch 100: MSE = 0.34
  epoch 200: MSE = 0.33    ← 不再下降了，收敛了，可以停
  epoch 201: MSE = 0.33

  这里 MSE 既是损失函数又是评估指标，一石二鸟。

  训练完成后的评估（评价模型好坏）

  跟 day03 正规方程训练完之后一模一样：

  ┌──────┬──────────────┬────────────────────────────────────────────────────┐
  │ 指标 │     公式     │                       看什么                       │
  ├──────┼──────────────┼────────────────────────────────────────────────────┤
  │ MSE  │ 误差²的平均  │ 越小越好，有单位（平方），不直观                   │
  ├──────┼──────────────┼────────────────────────────────────────────────────┤
  │ RMSE │ √MSE         │ 越小越好，和 y 同单位，好解释                      │
  ├──────┼──────────────┼────────────────────────────────────────────────────┤
  │ MAE  │ |误差|的平均 │ 越小越好，对异常值不敏感                           │
  ├──────┼──────────────┼────────────────────────────────────────────────────┤
  │ R²   │ 1 - MSE/方差 │ 越接近 1                                           │
  │      │              │ 越好，最常用，表示模型解释了多少比例的变化         │
  └──────┴──────────────┴────────────────────────────────────────────────────┘

  不管用正规方程还是梯度下降训练的，评估方式完全一样。
  评估看的是结果（预测准不准），不关心你怎么训的。

  多一个梯度下降特有的评估：loss 曲线形状

  这是正规方程没有的，因为正规方程一步到位没有迭代过程：

  好的 loss 曲线：                    坏的 loss 曲线：

  loss                                loss
   |█                                  |█     █
   |█                                  | █   █ █
   | █                                 |  █ █   █
   |  █                                |   █     █
   |   ██                              |
   |     ████████ ← 走平=收敛          |    ← 震荡=学习率太大
   +──────────                         +──────────

  SGDRegressor 是什么

  就是用 SGD（随机梯度下降）训练的线性回归。

  LinearRegression()   → 内部用正规方程（矩阵求逆），一步出结果
  SGDRegressor()       → 内部用梯度下降（迭代更新 w 和 b），逐步逼近结果

  两者解决的是同一个问题（线性回归），得到的结果几乎一样，只是求解方法不同：

  from sklearn.linear_model import LinearRegression, SGDRegressor

  # 正规方程版
  model1 = LinearRegression()
  model1.fit(X_train, y_train)          # 一步算完

  # 梯度下降版
  model2 = SGDRegressor(
      max_iter=1000,     # 最多迭代1000次
      eta0=0.01,         # 初始学习率
      learning_rate='invscaling'  # 学习率自动衰减
  )
  model2.fit(X_train, y_train)          # 迭代1000次逼近

  训练完之后：

  # 评估方式完全一样
  model1.score(X_test, y_test)   # → R² = 0.847
  model2.score(X_test, y_test)   # → R² = 0.845  ← 几乎一样

  什么时候用 SGDRegressor？

  特征数 n < 10000    → LinearRegression()，省事
  特征数 n > 10000    → SGDRegressor()，快
  数据是流式的        → SGDRegressor().partial_fit()，增量学习

  SGDRegressor 就是 sklearn 帮你包装好的梯度下降线性回归， 你不用自己写那 6 行
  numpy 代码，它全封装好了，连学习率衰减、标准化提醒都帮你做了。

> 讲一下回归模型评估方法吧 有哪几种方法？ 都是对w和d进行评估的吗 ？             

● 回归模型评估不是评估 w 和 b 本身，而是评估预测值 ŷ 和真实值 y 的差距。

  你不关心 w 是 2.3 还是 2.7，你只关心：用这组 w 和 b 算出来的预测准不准。

  评估指标

  1. MSE（均方误差）

           1   m
  MSE  =  ─── · Σ (ŷᵢ - yᵢ)²
           m  i=1

  误差平方再取平均

  - 平方会放大大误差 → 对离谱的预测惩罚更重
  - 单位是原始单位的平方（房价万元 → 万元²），不直观
  - 梯度下降的损失函数就是它，训练和评估用同一个东西

  2. RMSE（均方根误差）

  RMSE = √MSE

  - 就是 MSE 开个根号，单位回到原始单位
  - 预测房价，RMSE=3 意味着平均偏差约 3 万元，好解释
  - 实际汇报最常用的指标

  3. MAE（平均绝对误差）

           1   m
  MAE  =  ─── · Σ |ŷᵢ - yᵢ|
           m  i=1

  误差取绝对值再取平均

  - 不平方，大误差和小误差一视同仁
  - 数据有异常值时比 MSE 更稳

  4. R²（决定系数）

                MSE          Σ(ŷᵢ - yᵢ)²
  R²  =  1 - ──────  =  1 - ──────────────
               Var           Σ(yᵢ - ȳ)²

  ȳ = y 的平均值

  - 唯一一个有标准刻度的指标
  - R²=1 → 完美预测
  - R²=0 → 跟直接猜平均值一样差
  - R²<0 → 比猜平均值还差（模型有问题）
  - 不受单位影响，不同数据集之间可以比较

  对比

                单位      对异常值    直觉性    适合场景
  MSE           y²        敏感        差       损失函数 / 训练过程
  RMSE          y         敏感        好       汇报结果
  MAE           y         不敏感      好       有异常值时
  R²            无        敏感        最好     比较不同模型 / 不同数据集

  用具体数字感受

  真实房价 y  = [10,  20,  30,  40,  50]  万元
  预测房价 ŷ  = [12,  18,  33,  38,  55]  万元
  误差        = [ 2,  -2,   3,  -2,   5]

  MSE  = (4 + 4 + 9 + 4 + 25) / 5 = 9.2    万元²  ← 不直观
  RMSE = √9.2 = 3.03                        万元   ← 平均偏差约3万
  MAE  = (2 + 2 + 3 + 2 + 5) / 5  = 2.8    万元   ← 平均偏差约2.8万
  R²   = 1 - 9.2/200 = 0.954                       ← 解释了95.4%的变化

  RMSE 比 MAE 大，因为那个误差=5 的样本被平方放大了。

  评估方法（怎么用这些指标）

  指标定了，还要决定在哪些数据上算：

  留出法（Hold-out）

  全部数据
  ├── 训练集 80%  → fit()  → 训练模型
  └── 测试集 20%  → score() → 算 R²/MSE

  train_test_split(X, y, test_size=0.2)

  最简单，day03 你已经用过了。

  K 折交叉验证（K-Fold CV）

  把数据分成 K=5 份，每次用 1 份测试、4 份训练，轮 5 次：

  第1轮：[测试][训练][训练][训练][训练] → R²= 0.82
  第2轮：[训练][测试][训练][训练][训练] → R²= 0.87
  第3轮：[训练][训练][测试][训练][训练] → R²= 0.84
  第4轮：[训练][训练][训练][测试][训练] → R²= 0.81
  第5轮：[训练][训练][训练][训练][测试] → R²= 0.86

  最终 R² = 平均 = 0.84

  - 每个样本都当过一次测试集，评估更稳
  - 留出法可能碰巧切到好的/坏的数据，交叉验证避免了这个运气成分

  from sklearn.model_selection import cross_val_score
  scores = cross_val_score(model, X, y, cv=5, scoring='r2')
  print(scores.mean())   # → 0.84

  怎么选

  数据量大（>10000）  → 留出法就够，快
  数据量小（<5000）   → 交叉验证，结果更可信
  调超参数（选学习率） → 交叉验证，避免过拟合到某一次划分

  总结

  评估指标（算什么）：MSE、RMSE、MAE、R²  → 评的是预测准不准，不是 w 和 b 的值
  评估方法（怎么算）：留出法、交叉验证      → 决定在哪些数据上算指标

> 这个预测值和真实值 是重新拿我们调整好的公式进行计算的?                        
那我们观测点曲线又是什么曲线呢                                                  

● 预测值怎么来的

  对，就是拿训练好的 w 和 b 代入公式，在测试集上算：

  训练阶段：梯度下降跑完 → 得到 w=2.3, b=1.5

  评估阶段：
    拿测试集的 x 代入：ŷ = 2.3 × x + 1.5

    测试样本1: x=3  → ŷ = 2.3×3 + 1.5 = 8.4    真实 y=8    误差=0.4
    测试样本2: x=5  → ŷ = 2.3×5 + 1.5 = 13.0   真实 y=14   误差=-1.0
    测试样本3: x=7  → ŷ = 2.3×7 + 1.5 = 17.6   真实 y=18   误差=-0.4

    MSE = (0.16 + 1.0 + 0.16) / 3 = 0.44
    R²  = ...

  这些测试数据在训练时模型从没见过，所以能真实反映模型的泛化能力。

  两条曲线完全不同

  你说的 loss 曲线和评估是两个东西，别混了：

  loss 曲线（训练过程中画的）

  横轴是迭代次数，纵轴是训练集上的 MSE：

  loss
   |
  40|█
    |█
  30|█
    | █
  20|  █
    |   █
  10|    ██
    |      █████████████  ← 走平=收敛
   0+─────────────────────
    0  50  100  200  500   迭代次数(epoch)

  这条曲线的作用：判断梯度下降有没有正常工作（学习率对不对、该不该停）

  每个点怎么来的：
  epoch 1:   用当前 w,b 对训练集所有样本算一遍 MSE → 画一个点
  epoch 2:   更新 w,b 后再算一遍 MSE → 画一个点
  epoch 3:   再更新再算 → 画一个点
  ...

  评估指标（训练结束后算的）

  训练跑完了，w 和 b 定了，在测试集上算一次
  R²、MSE、RMSE，就一个数字，不是曲线。

  训练结束 → w=2.3, b=1.5 → 在测试集上 → R²=0.85 → 完了

  关系

  训练过程                              训练结束后
  ──────                              ────────

  loss 曲线                            评估指标
  ├── 在训练集上算                      ├── 在测试集上算
  ├── 每个 epoch 算一次 → 画成曲线       ├── 只算一次 → 一个数字
  ├── 目的：学习率对不对                 ├── 目的：模型好不好
  └── 看的是"训练过程顺不顺利"           └── 看的是"最终结果准不准"

  先保证 loss 曲线正常收敛
               ↓
         再看评估指标才有意义

  如果 loss 曲线还在震荡就去看 R²，那个 R²
  是没意义的——模型还没训好，评估个寂寞。

> 我看懂了 这个LOSS 那么三种评估标准有什么区别呢 是更精细化的评估？             

● 不是精细化的关系，是看问题的角度不同。

  MSE、RMSE、MAE、R² 的区别

  四个指标算的都是同一批预测结果，只是换了不同的"打分方式"：

  同一组预测结果：
  真实 y  = [10, 20, 30, 40, 50]
  预测 ŷ  = [12, 18, 33, 38, 55]
  误差    = [ 2, -2,  3, -2,  5]

  MSE — 放大大误差

  (4 + 4 + 9 + 4 + 25) / 5 = 9.2

  平方了，那个误差=5 的样本贡献了 25，占了一半以上
  → 对"离谱的预测"惩罚很重
  → 单位是万元²，没法直接跟 y 比

  RMSE — MSE 开根号，恢复单位

  √9.2 = 3.03 万元

  → "平均偏差大约 3 万" ← 一句话就能跟别人说清楚
  → 还是对大误差敏感（因为骨子里还是 MSE）

  MAE — 不放大，一视同仁

  (2 + 2 + 3 + 2 + 5) / 5 = 2.8 万元

  → 每个误差按原值算，不平方
  → 那个误差=5 的样本只贡献 5，不是 25
  → 数据有异常值（比如一套房标错价）时更稳

  R² — 跟"瞎猜"比你好多少

  R² = 0.954

  → 不看具体偏差，看"你比无脑猜平均值好多少"
  → 0.95 意思是：模型解释了 95% 的变化规律，只剩 5% 没抓住
  → 无单位，不同数据集之间可以比

  什么时候用哪个

  场景                              选哪个         为什么
  ──────                           ──────         ──────

  R² + RMSE     ← 默认组合，90% 的情况够用

  加 MAE        ← 数据有异常值时，跟 RMSE 对比
                   RMSE 比 MAE 大很多 → 说明有少数离谱的预测
                   RMSE 和 MAE 接近   → 误差比较均匀

  MSE           ← 基本不单独看，就是训练时的损失函数
                   RMSE = √MSE，看 RMSE 就行

  总结

  训练过程中：看 loss 曲线（MSE）→ 梯度下降正不正常
  训练完之后：看 R² + RMSE       → 模型好不好、偏差多大
  有异常值时：加看 MAE           → 跟 RMSE 对比判断异常影响
  选模型时  ：看测试集的 R²      → 但最终是人做决定

  评估这块就这些了，没有更复杂的东西。继续聊标准化？

> R平方是什么                                                                   

● 就是你前面看到的那个 R²，展开讲一下它到底在算什么。

  R² 的核心思想

  你的模型比"无脑猜平均值"好多少？

  假设房价 y = [10, 20, 30, 40, 50]，平均值 ȳ = 30

  最蠢的预测：每套房都猜 30
    误差² = (10-30)² + (20-30)² + (30-30)² + (40-30)² + (50-30)²
         = 400 + 100 + 0 + 100 + 400
         = 1000                          ← 这叫 SS_total（总变差）

  你的模型预测：ŷ = [12, 18, 33, 38, 55]
    误差² = (12-10)² + (18-20)² + (33-30)² + (38-40)² + (55-50)²
         = 4 + 4 + 9 + 4 + 25
         = 46                            ← 这叫 SS_residual（残差）

  R² 就是比一下：

           SS_residual       46
  R² = 1 - ──────────── = 1 - ──── = 1 - 0.046 = 0.954
           SS_total          1000

  你的模型把误差从 1000 降到了 46，消除了 95.4% 的误差。

  直觉理解

  R² = 1.0  → 你的模型完美预测，残差 = 0
  R² = 0.95 → 消除了 95% 的误差，只剩 5% 没解释
  R² = 0.5  → 只消除了一半误差，跟瞎猜比好一半
  R² = 0    → 跟猜平均值一样差
  R² < 0    → 比猜平均值还差（模型有严重问题）

  为什么叫 R "平方"

  因为在简单线性回归里，R² 恰好等于相关系数 r 的平方：

  相关系数 r = 0.97（x 和 y 的线性相关程度）
  R² = 0.97² = 0.94

  r 看的是"x 和 y 有多相关"
  R² 看的是"模型解释了多少变化"
  简单线性回归里两者等价，多元回归里只看 R²

  就这些，没有更深的了。R² 就是拿你的模型跟"猜平均值"比，看你好了百分之多少。

> 这和MAE ME RMSE的区别是？                                                     

● 它们算的东西不一样：

  MSE / RMSE / MAE  → 算的是"误差有多大"（一个绝对数字）
  R²                → 算的是"比瞎猜好多少"（一个比例）

  用同一组数据看区别：

  真实 y = [10, 20, 30, 40, 50]
  预测 ŷ = [12, 18, 33, 38, 55]
  误差   = [ 2, -2,  3, -2,  5]

  MSE  = 9.2       ← 误差平方的平均（万元²）
  RMSE = 3.03      ← 平均偏差约 3 万元
  MAE  = 2.8       ← 平均偏差约 2.8 万元
  R²   = 0.954     ← 比猜平均值好了 95.4%

  MSE/RMSE/MAE 告诉你"偏了多少"，但你不知道这个数字算好还是坏：

  RMSE = 3 万

  房价 500 万的场景 → 偏 3 万？太准了
  房价 10 万的场景  → 偏 3 万？太离谱了

  同样的 RMSE，好不好完全取决于 y 的范围

  R² 不需要看 y 的范围就能判断：

  R² = 0.95 → 不管预测的是房价还是气温，都代表"解释了 95% 的变化"

  所以它们是配合使用的：

  R²   = 0.95  → "模型不错"（整体判断）
  RMSE = 3 万  → "具体偏差 3 万，能不能接受？"（业务判断）

  R² 给定性判断，RMSE 给定量感受，各管各的。

> 梯度下降的标准化有什么说法吗 和正规方程的标准化不一样吗                       

● 标准化本身是同一个操作，都是把特征缩放到同一量级。区别在于正规方程不需要标准化
  ，梯度下降必须标准化。

  为什么正规方程不需要

  正规方程是直接解方程，一步到位：

  w = (XᵀX)⁻¹Xᵀy

  不管 x₁ 是 1~5 还是 1000~5000
  矩阵求逆都能算出正确的 w

  为什么梯度下降必须标准化

  因为梯度 = 误差 × 特征值，特征值大的方向梯度就大，步子就大：

  x₁ = 面积（1~5）
  x₂ = 收入（10000~50000）

  同样的误差 = -2：
    dw₁ = -2 × 3      = -6
    dw₂ = -2 × 30000  = -60000

  w₂ 的梯度比 w₁ 大一万倍
  同一个学习率 α = 0.01：
    w₁ 更新 0.06     ← 几乎没动
    w₂ 更新 600      ← 飞出去了

  不标准化的下降路径：

  w₂
   |
   |  \  /\  /\  /
   |   \/  \/  \/     ← 在 w₂ 方向来回震荡
   |         \  /         因为步子太大
   |          \/
   |           \
   |            ★ 谷底
   +──────────────── w₁
       w₁ 方向几乎没动，因为步子太小

  标准化之后：

  x₁ 和 x₂ 都变成 0 附近，范围差不多

  dw₁ = -2 × 0.3  = -0.6
  dw₂ = -2 × 0.5  = -1.0

  两个方向步子差不多

  w₂
   |
   |  \
   |   \
   |    \
   |     ★ 谷底      ← 直奔谷底，不震荡
   +──────────── w₁

  标准化方法也是一样的

  from sklearn.preprocessing import StandardScaler

  scaler = StandardScaler()
  X_train_scaled = scaler.fit_transform(X_train)
  X_test_scaled  = scaler.transform(X_test)      # 用训练集的均值和标准差

  跟正规方程用的是同一个工具，同一个操作。

  总结

                    正规方程          梯度下降
  标准化操作         一样              一样
  需不需要做         不需要            必须做
  不做会怎样         结果一样正确      震荡、不收敛、或收敛极慢
  原因              直接解方程，       梯度 = 误差 × 特征值，
                    不受量级影响       量级不同步子就不均匀

  一句话：标准化不是为了改变数据，是为了让梯度下降各方向走得一样快。

● # ═══════════════════════════════════════════
  # 正规方程 vs 梯度下降 完整对比
  # ═══════════════════════════════════════════

  import numpy as np
  from sklearn.model_selection import train_test_split
  from sklearn.preprocessing import StandardScaler
  from sklearn.linear_model import LinearRegression, SGDRegressor
  from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

  # ─── 造数据 ───
  np.random.seed(42)
  X = np.random.rand(100, 3) * [10, 5000, 200]   # 3个特征，量级差很大
  #    特征1: 0~10    特征2: 0~5000    特征3: 0~200
  y = 3*X[:,0] + 0.01*X[:,1] + 0.5*X[:,2] + np.random.randn(100)*2

  X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)


  # ═══════════════════════════════════════════
  # 方法1：正规方程（不需要标准化）
  # ═══════════════════════════════════════════

  model1 = LinearRegression()
  model1.fit(X_train, y_train)              # 一步算完
  y_pred1 = model1.predict(X_test)

  print("═══ 正规方程 ═══")
  print(f"w = {model1.coef_}")
  print(f"b = {model1.intercept_:.4f}")
  print(f"R²   = {r2_score(y_test, y_pred1):.4f}")
  print(f"RMSE = {np.sqrt(mean_squared_error(y_test, y_pred1)):.4f}")
  print(f"MAE  = {mean_absolute_error(y_test, y_pred1):.4f}")


  # ═══════════════════════════════════════════
  # 方法2：梯度下降（必须标准化）
  # ═══════════════════════════════════════════

  # 标准化 ← 唯一多出来的步骤
  scaler = StandardScaler()
  X_train_scaled = scaler.fit_transform(X_train)   # 训练集：算均值标准差 + 缩放
  X_test_scaled  = scaler.transform(X_test)         # 测试集：只缩放，不重新算

  model2 = SGDRegressor(
      max_iter=1000,          # 最多迭代1000次
      eta0=0.01,              # 初始学习率
      learning_rate='invscaling',  # 学习率自动衰减
      random_state=42
  )
  model2.fit(X_train_scaled, y_train)       # 迭代逼近
  y_pred2 = model2.predict(X_test_scaled)

  print("\n═══ 梯度下降 ═══")
  print(f"w = {model2.coef_}")
  print(f"b = {model2.intercept_[0]:.4f}")
  print(f"R²   = {r2_score(y_test, y_pred2):.4f}")
  print(f"RMSE = {np.sqrt(mean_squared_error(y_test, y_pred2)):.4f}")
  print(f"MAE  = {mean_absolute_error(y_test, y_pred2):.4f}")

    → 正规方程算不动 → 用梯度下降
    → 这些特征是真实的，不是你造的

  场景2：原始特征很少，你加多项式把它变多了
    比如 8 个特征，degree=4 变成 495 个
    → 这是你自己造出来的，没必要
    → 换模型就行了

  所以实际路径是这样的

  原始特征少（1~20个）：
    ├── 正规方程跑一下 → R² 够了 → 结束
    ├── R² 不够 → 加多项式（degree=2~3）→ 够了 → 结束
    └── 还不够 → 不继续加多项式
               → 换非线性模型（随机森林、XGBoost）
                 这些模型内部自己处理非线性
                 不需要你手动造 x² x³ x⁴

  原始特征多（几百上千个）：
    ├── 正规方程算不动
    ├── 梯度下降上场 → 跑线性回归
    ├── R² 不够 → 不加多项式（会爆炸）
    └── 直接换模型

  关键区别

  梯度下降解决的问题：特征多时怎么"算得动"
  多项式解决的问题：  关系是弯的时怎么"拟合得好"

  两个问题不在一个层面上：

    "算不算得动" → 正规方程 vs 梯度下降（求解方法）
    "拟合得好不好" → 一次 vs 多项式 vs 换模型（模型选择）

  梯度下降不是为了加多项式才用的，是为了原始特征就很多的时候能算得动线性回归。

● 拟合

  拟合就三种状态：

  欠拟合                    正好                     过拟合
  (underfitting)           (good fit)               (overfitting)

  y                        y                        y
  |    ·  ·                |    ·  ·                |    ·  ·
  |  ·      ·              |  ·  ╲   ·              |  · ╱╲ ╱·
  | ·────────·             | · ╱   ╲  ·             | ·╱  ╲╱  ╲·
  |·          ·            |·╱      ╲  ·            |╱╱        ╲·
  +──────────── x          +──────────── x          +──────────── x

  直线硬拟合弯曲数据         抓住了主要趋势             每个点都贴上去了
  训练集差，测试集也差       训练集好，测试集也好         训练集完美，测试集差

  用数字看：

                  训练集 R²    测试集 R²    诊断
  欠拟合            0.50        0.48      都差 → 模型太简单
  正好              0.90        0.88      都好，差距小 → 刚好
  过拟合            0.99        0.65      训练好测试差 → 模型太复杂

  判断方法就一个：比较训练集和测试集的 R²

  两个都低          → 欠拟合 → 加复杂度（多项式/换模型）
  两个都高且接近    → 正好   → 不用动
  训练高测试低      → 过拟合 → 减复杂度

  过拟合为什么会发生

  模型太复杂，把噪声也当成规律学了：

  真实数据 = 规律 + 噪声

  真实规律：y = 2x + 1
  实际数据：y = 2x + 1 + 随机波动

  简单模型：学到 y = 2.1x + 0.9     ← 抓住了规律，忽略了噪声 ✓
  复杂模型：学到 y = 0.3x⁴ - 2x³ + 5x² - 3x + 1.2
                                      ← 连噪声都拟合了 ✗

  复杂模型在训练集上分数很高，但遇到新数据（噪声不同）就崩了。

  正则化 — 专门治过拟合

  过拟合的特征是参数 w 太大。模型为了贴合每个点，w 会变得很极端：

  正好的模型：  y = 2.1x₁ + 0.8x₂ + 1.0        ← w 都不大
  过拟合模型：  y = 150x₁ - 230x₂ + 85x₃ + ...  ← w 极端大

  正则化的思路：在损失函数里加一个惩罚项，w 越大惩罚越重

  原始损失函数：     L = MSE
  正则化后：        L = MSE + λ × (w 的大小)
                                ↑
                           这个惩罚项逼着 w 不能太大

  λ（lambda）控制惩罚力度：

  λ = 0     → 没有惩罚 → 跟原来一样 → 可能过拟合
  λ 适中    → 适度惩罚 → w 合理 → 刚好
  λ 太大    → 惩罚太重 → w 全被压成接近 0 → 欠拟合

  两种正则化方式

  L2 正则化（Ridge 岭回归）

  L = MSE + λ × Σwⱼ²

  惩罚 w 的平方和
  效果：把所有 w 都变小，但不会变成 0
        每个特征都保留，只是影响力减弱

  from sklearn.linear_model import Ridge

  model = Ridge(alpha=1.0)   # alpha 就是 λ
  model.fit(X_train, y_train)

  L1 正则化（Lasso）

  L = MSE + λ × Σ|wⱼ|

  惩罚 w 的绝对值之和
  效果：会把不重要的 w 直接压成 0
        自动帮你选特征——w=0 的特征等于被丢弃了

  from sklearn.linear_model import Lasso

  model = Lasso(alpha=1.0)
  model.fit(X_train, y_train)
  # 训练完看 model.coef_，很多是 0

  对比

                  Ridge (L2)              Lasso (L1)
  惩罚项          Σwⱼ²                    Σ|wⱼ|
  w 的变化        全部变小，不会变成0       不重要的直接变成0
  特征选择        不做                     自动做
  适用场景        所有特征都有用            怀疑有些特征是噪声

  train_r2 = model.score(X_train, y_train)

    0.90        0.88       小        正好（都高）
    0.99        0.65       大        过拟合（训练高测试低）

  就比两个数字，不需要画图肉眼判断。

  三种状态分别怎么操作

  欠拟合（训练集和测试集都低）

  模型太简单，学不到规律。

  操作1：加多项式特征（特征少的时候）
    PolynomialFeatures(degree=2)
    → 让模型能拟合弯曲关系

  操作2：加更多特征
    → 原始数据里还有没用上的特征？加进来

  操作3：换更复杂的模型
    LinearRegression → RandomForest / XGBoost
    → 让模型本身更强

  操作4：减小正则化强度
    Ridge(alpha=10) → Ridge(alpha=0.1)
    → 正则化太强会压制模型能力，放松一点

  正好（都高，差距小）

  不用动，部署上线

  过拟合（训练高，测试低）

  模型太复杂，把噪声也学了。

  操作1：加正则化
    LinearRegression → Ridge(alpha=1.0) 或 Lasso(alpha=0.1)
    → 惩罚大的 w，防止模型太极端

  操作2：减少特征
    degree=4 → degree=2
    → 或者用 Lasso 自动把没用的特征压成 0

  操作3：加数据量
    样本太少时模型容易过拟合
    → 数据多了，噪声被平均掉，过拟合自然缓解

  操作4：换更简单的模型
    → 多项式换回线性
    → 复杂模型换回简单模型

  总结

  判断方式：比 训练集R² 和 测试集R²

  欠拟合（都低）        → 加复杂度：加特征、加多项式、换模型、减正则化
  正好  （都高差距小）  → 不动
  过拟合（训练高测试低）→ 减复杂度：加正则化、减特征、加数据、换简单模型

  方向永远是：
    欠拟合 → 往复杂方向调
    过拟合 → 往简单方向调
    调到两个 R² 都高且接近为止

● R² 怎么算的

  不是对平均值的差值，是两组误差的比较：

  训练集/测试集上：

  第一组：你的模型预测的误差
    Σ(ŷᵢ - yᵢ)²   ← 每个预测值减真实值，平方，累加

  第二组：无脑猜平均值的误差
    Σ(ȳ - yᵢ)²    ← 平均值减真实值，平方，累加

  R² = 1 - 第一组/第二组

  具体算：

  y 真实值 = [10, 20, 30, 40, 50]    平均值 ȳ = 30
  ŷ 预测值 = [12, 18, 33, 38, 55]

  你的模型误差²：
    (12-10)² + (18-20)² + (33-30)² + (38-40)² + (55-50)²
  = 4 + 4 + 9 + 4 + 25 = 46

  猜平均值误差²：
    (30-10)² + (30-20)² + (30-30)² + (30-40)² + (30-50)²
  = 400 + 100 + 0 + 100 + 400 = 1000

  R² = 1 - 46/1000 = 0.954

  训练集 R² 和测试集 R² 是分别算的，各自用各自的数据：

  # 训练集 R²：用训练数据预测，跟训练数据的真实值比
  train_pred = model.predict(X_train)
  train_r2 = r2_score(y_train, train_pred)

  # 测试集 R²：用测试数据预测，跟测试数据的真实值比
  test_pred = model.predict(X_test)
  test_r2 = r2_score(y_test, test_pred)

  为什么要有正则化

  从一个现象说起。你用多项式拟合加州房价：

  # degree=2
  poly = PolynomialFeatures(degree=2)
  model.fit(X_train_poly, y_train)

  看一下训练出来的 w：

  无多项式：    w = [0.4, 0.01, -0.1, 0.6, ...]     ← w 都不大
  degree=2：   w = [0.4, 0.01, -0.1, 0.6, ...]     ← 原始特征还行
               w_新 = [15.3, -28.7, 42.1, -33.5, ...] ← 交叉项的 w 爆了
  degree=4：   w = [350, -1200, 4500, -8900, ...]   ← 全部爆炸

  模型越复杂，w 越极端。 为什么？

  训练集有 100 个点
  degree=4 有 495 个参数

  为了贴合每一个点，模型拼命调 w
  这边 +4500 那边 -8900，疯狂拉扯

  结果就是：
    训练集上每个点都贴得很紧 → R² = 0.99
    但曲线在两个点之间疯狂震荡 → 新数据来了就崩

  y
  |         训练点
  |    ·    ↓    ·
  |  ·  ╱╲·╱╲  ·  ╱╲     ← w 太大，曲线剧烈震荡
  | ·╱╱  ╲╱  ╲╱ ·╱  ╲       每个训练点都贴上了
  |╱              ╲          但中间的预测完全离谱
  +──────────────────── x

  正则化就是一句话：别让 w 太大。

  原来：模型只管一件事
    → 让预测值尽量接近真实值（最小化 MSE）
    → w 想多大就多大，只要 MSE 低

  加了正则化：模型要管两件事
    → 让预测值接近真实值（MSE 要小）
    → 同时 w 不能太大（惩罚项要小）
    → 两个目标之间取平衡

  L = MSE + λ × Σwⱼ²
      ↑          ↑
    预测要准    w 不能大

    λ 小 → 主要管预测准 → 接近无正则化
    λ 大 → 主要管 w 小  → w 被压制，模型变简单

  效果：

  无正则化：w = [350, -1200, 4500, -8900]
    → 曲线疯狂震荡 → 过拟合

  加正则化：w = [2.1, -0.8, 1.5, -0.3]
    → w 被压小了 → 曲线平滑了 → 不过拟合了

  正则化存在的唯一原因就是防止过拟合。 如果你的模型没有过拟合（训练集和测试集 R²
   接近），不需要加正则化。

> 正则化是如何影响学习率变更的？                                                

● 正则化不影响学习率本身，学习率 α 还是那个 α。它影响的是梯度的大小。

  没有正则化时

  dw = 1/m × Σ(ŷᵢ - yᵢ) × xᵢ

  w = w - α × dw

  梯度只看误差。

  加了 L2 正则化（Ridge）后

  dw = 1/m × Σ(ŷᵢ - yᵢ) × xᵢ + λ × w
                                   ↑
                             多了这一项

  w = w - α × (原来的梯度 + λ × w)

  就多了一个 λ × w，看看它干了什么：

  假设当前 w = 50（太大了），α = 0.1，λ = 0.1

  无正则化：
    dw = -3（只看误差）
    w = 50 - 0.1 × (-3) = 50.3      ← w 还在变大

  有正则化：
    dw = -3 + 0.1 × 50 = -3 + 5 = 2
                           ↑
                     w 越大，这个惩罚越大
    w = 50 - 0.1 × 2 = 49.8         ← w 被拉回来了

  w 小的时候：
    λ × w 很小 → 惩罚几乎没有 → 跟没加正则化一样

  w 大的时候：
    λ × w 很大 → 梯度方向被扭转 → w 被往 0 拉

    梯度说"w 该变大" ←──对抗──→ 正则化说"w 该变小"
                      ↑
                最终 w 停在一个平衡点
                不会太大也不会太小

  用迭代过程看

  无正则化，w 的变化：
    0 → 5 → 15 → 35 → 80 → 150 → 350    ← 越来越大，没人管

  有正则化，w 的变化：
    0 → 5 → 12 → 18 → 20 → 20.1 → 20.1  ← 到 20 就涨不动了
                                              因为 λ×w 的惩罚跟误差梯度平衡了

  所以正则化和学习率的关系

  学习率 α：控制每步走多远（步长）
  正则化 λ：控制 w 能长到多大（天花板）

  两个各管各的，不互相影响：

  α 大 → 走得快，但 w 的天花板不变
  α 小 → 走得慢，但 w 的天花板不变
  λ 大 → w 的天花板低（模型更简单）
  λ 小 → w 的天花板高（模型更自由）

  调参时两个分开调：

  先调 α → 看 loss 曲线正常收敛
  再调 λ → 看训练集和测试集 R² 的差距
           差距大 → λ 调大
           都低了 → λ 调小

  这块清楚了吗？

> 所有这个正则化就是告诉模型的w 它原来在哪个方向 应该往哪里走吗                 
就相当于地心引力会把向上的东西往下拉一样                                        

● 这个比喻非常到位。

  没有正则化：                    有正则化：

  w                               w
  |                               |
  |            🚀 w飞出去了        |         ← 天花板（λ 决定高度）
  |           ↑                   |    ·──── w 到这就上不去了
  |          ↑                    |   ↑ ↓重力
  |         ↑                     |  ↑  ↓
  |        ↑                      | ↑
  |       ↑  误差一直推着w往上     |↑  误差往上推
  |      ↑   没人管               |   正则化往下拉
  +──────── 迭代次数              +──────── 迭代次数

  就是地心引力：

  误差梯度 = 向上的推力（"w 该更大，预测才准"）
  正则化   = 向下的引力（"w 别太大"）

  w 小的时候：引力弱，推力大 → w 往上走
  w 大的时候：引力强，推力弱 → 走不动了
  最终平衡：  推力 = 引力    → w 稳定在一个合理的值

  λ 大 = 引力强 = w 飞不高
  λ 小 = 引力弱 = w 能飞高一点
  λ = 0 = 没有引力 = w 随便飞

  而且跟重力一样，离地面越远引力效果越明显：

  w = 1:    正则化惩罚 = λ × 1   = 很小，几乎不拉
  w = 50:   正则化惩罚 = λ × 50  = 明显在拉了
  w = 500:  正则化惩罚 = λ × 500 = 使劲往回拽

> 所以正则化只是在dw阶段和评估Loss介入并影响学习出来的公式吗                    

● 对，就介入两个地方：

  正向传播：  ŷ = w·x + b          ← 不变，该怎么预测还怎么预测

  计算 loss：  L = MSE + λ×Σwⱼ²    ← 介入1：loss 多了惩罚项
                         ↑
                      多了这个

  反向传播：  dw = 误差梯度 + λ×w   ← 介入2：梯度多了一项
                             ↑
                          多了这个

  更新参数：  w = w - α × dw       ← 不变，还是同样的更新公式

  只在 loss 和 dw 这两处加了东西，其他全部不变。

  而且这两处加的东西本质上是同一件事——loss 里加了 λ×Σwⱼ²，对 w 求导自然就多出
  λ×w。不是分别设计的两个机制，是一个因（改了 loss）导致的一个果（梯度变了）。

  改 loss 函数
      ↓ 求导
  梯度自动变了
      ↓ 更新
  w 自动变小了
      ↓ 预测
  最终模型更平滑

  训练完之后呢？

  训练完之后：
    拿到 w 和 b → 预测 ŷ = w·x + b → 评估 R²/RMSE

    这一步正则化完全不参与
    正则化只在训练过程中起作用
    训练完了它的任务就结束了

  正则化只管训练过程，不管训练结果怎么用。
  就像地心引力只在球上升过程中拉它，球落地之后引力的任务就完成了。

> 误差梯度怎么计算 让我们复习一下?                                              

● 从头走一遍，就一个样本：

  数据：x=2, y=7
  当前：w=1, b=3, α=0.1

  ═══ 正向传播 ═══

  ŷ = w·x + b = 1×2 + 3 = 5
  误差 = ŷ - y = 5 - 7 = -2
  loss = 误差² = 4

  ═══ 反向传播（链式法则）═══

  L = (w·x + b - y)²

  拆成三层：
    z = w·x + b = 5        ← 预测值
    e = z - y = -2          ← 误差
    L = e² = 4              ← 损失

  从外往里求导，每层乘起来：

    ∂L/∂e = 2e = 2×(-2) = -4
    ∂e/∂z = 1
    ∂z/∂w = x = 2
    ∂z/∂b = 1

  dw = -4 × 1 × 2 = -8
  db = -4 × 1 × 1 = -4

  ═══ 更新参数 ═══

  w = 1 - 0.1×(-8) = 1.8
  b = 3 - 0.1×(-4) = 3.4

  多个样本就是每个算一遍，取平均：

  样本1: x=2, y=7  → dw₁=-8,  db₁=-4
  样本2: x=1, y=3  → dw₂=-2,  db₂=-2
  样本3: x=3, y=10 → dw₃=-15, db₃=-5

  dw = (-8 + -2 + -15) / 3 = -8.33
  db = (-4 + -2 + -5)  / 3 = -3.67

  w = w - 0.1 × (-8.33)
  b = b - 0.1 × (-3.67)

  加了正则化的话就多一项：

  无正则化：dw = -8.33
  有正则化：dw = -8.33 + λ×w    ← 就多了这个
                        ↑
                    把 w 往 0 拉

  就这些，没有别的了。

> 那么 L1正则化和L2正则化有什么区别呢                                           

● 区别就在惩罚项怎么算：

  L2（Ridge）— 惩罚 w 的平方

  L = MSE + λ × (w₁² + w₂² + w₃²)

  dw₁ = 误差梯度 + λ × 2w₁
                         ↑
                   w 越大惩罚越猛，但永远不会把 w 压到 0

  w 的变化过程：

  w = 50 → 30 → 18 → 11 → 7 → 4.5 → 3 → 2 → 1.5 → 1.2 ...
                                                      ↑
                                                越来越小但不到 0
                                                像弹簧，越拉越弱

  L1（Lasso）— 惩罚 w 的绝对值

  L = MSE + λ × (|w₁| + |w₂| + |w₃|)

  dw₁ = 误差梯度 + λ × sign(w₁)
                         ↑
                   不管 w 多大多小，每次都固定拉 λ 这么多
                   sign(w) = w>0 时+1，w<0 时-1

  w 的变化过程：

  w = 50 → 30 → 18 → 11 → 7 → 4.5 → 3 → 1.5 → 0 ← 直接到 0 了
                                                     被砍没了

  为什么一个到 0 一个不到 0

  L2 的力 = λ × 2w        L1 的力 = λ × 1（固定）

  w=50 时：                w=50 时：
    L2 拉力 = 100（猛）      L1 拉力 = 1
  w=1 时：                 w=1 时：
    L2 拉力 = 2（弱了）      L1 拉力 = 1（还是一样猛）
  w=0.01 时：              w=0.01 时：
    L2 拉力 = 0.02（几乎没了）L1 拉力 = 1（照砍不误）

  L2 像弹簧：              L1 像刀：
    w 大 → 拉力大            不管 w 多大多小
    w 小 → 拉力也小          每次砍固定的量
    → 永远拉不到 0           → 小的 w 直接被砍到 0

  效果对比

  假设 5 个特征的 w：

  无正则化：w = [15.0,  0.3,  8.0,  0.1,  12.0]
                        ↑          ↑
                      这两个本来就不重要

  L2 Ridge：w = [10.2,  0.2,  5.5,  0.07,  8.1]
                全部变小了，但都还在

  L1 Lasso：w = [12.0,  0.0,  6.8,  0.0,  10.0]
                        ↑          ↑
                     直接变成 0 了 = 这个特征被丢弃了

  什么时候用哪个

  L2 Ridge：
    所有特征都可能有用，只是不想让 w 太极端
    → 全部保留，全部压小

  L1 Lasso：
    怀疑有些特征是噪声/没用的
    → 自动帮你选特征，没用的 w 直接归 0
    → 训练完看 w，哪些是 0 就知道哪些特征没用

  不确定用哪个：
    → ElasticNet（两个一起用）

  from sklearn.linear_model import Ridge, Lasso, ElasticNet

  Ridge(alpha=1.0)       # L2，全部压小
  Lasso(alpha=0.1)       # L1，砍掉没用的
  ElasticNet(alpha=0.1)  # L1 + L2 混合

  一句话：L2 是弹簧拉所有人，L1 是刀砍弱者。

> 前面那个案例的 x2w1 和sin（w） 是自定义的？                                   

● 不是自定义的，是对惩罚项求导自然得到的：

  L2 的 2w 怎么来的

  惩罚项 = λ × w²

  对 w 求导：

  w² → 2w     ← 就是最基础的求导规则：xⁿ → n·xⁿ⁻¹

  所以 dw 多出来的项 = λ × 2w

  跟链式法则里 L = e² 求导得到 2e 完全一样的规则。

  L1 的 sign(w) 怎么来的

  惩罚项 = λ × |w|

  对 w 求导：

  |w| 的导数：
    w > 0 时：|w| = w   → 导数 = +1
    w < 0 时：|w| = -w  → 导数 = -1
    w = 0 时：不可导（尖角）

  把这个写成一个函数就叫 sign(w)：

  sign(w) = {  +1    w > 0
            {  -1    w < 0

  就是正负号函数：

       +1 ──────────
            |
  ──────── -1

          w=0

  所以两个都是求导的结果

  L2：λ × w²   求导 → λ × 2w       ← 力跟 w 成正比（弹簧）
  L1：λ × |w|  求导 → λ × sign(w)  ← 力是固定的 ±1（刀）

  不是谁规定的，是数学推出来的。改了 loss 函数，求导结果自然就变了。

> dw 也要对 正则项求导吗                                                        

● 对，因为 dw 是对整个 loss 函数求导，正则项是 loss 的一部分：

  原来：  L = MSE
  现在：  L = MSE + λ×w²
               ↑      ↑
             第一项   第二项（正则项）

  求导时每一项分别求导再加起来：

  dw = ∂L/∂w

     = ∂(MSE)/∂w + ∂(λ×w²)/∂w
          ↑              ↑
      误差梯度          2λw

     = 1/m × Σ(ŷᵢ-yᵢ)×xᵢ + 2λw

  就跟你学过的基础求导一样：

  f(x) = x³ + 5x²

  df/dx = 3x² + 10x
           ↑     ↑
         第一项  第二项
         分别求导再加起来

  loss 函数也是一样，不管里面有几项，每项各求各的导，最后加起来就是 dw。

> 那L1的sin又是怎么来的？                                                       

● 是 sign（符号函数），不是 sin（三角函数），两个不同的东西。

  sign 来自对 |w| 求导：

  |w| 就是绝对值，把负数变正数：

  |5|  = 5
  |-3| = 3
  |0|  = 0

  绝对值的图长这样：

  |w|
   |  ╲   ╱
   |   ╲ ╱
   |    ╲      ← w=0 这里有个尖角
   +──────── w
     -3  0  3

  两边分别求导：

  右边（w > 0）：|w| = w    → 导数 = 1
  左边（w < 0）：|w| = -w   → 导数 = -1

  合起来就叫 sign(w)：

  sign(3)  = +1    ← 正数的符号是 +
  sign(-3) = -1    ← 负数的符号是 -
  sign(0)  = 0     ← 零没有符号

  所以 L1 的正则项求导：

  ∂(λ × |w|)/∂w = λ × sign(w)

  w = 5 时：  λ × (+1) = +λ     ← 往负方向拉
  w = -5 时： λ × (-1) = -λ     ← 往正方向拉

  不管 w 多大多小，拉力都是固定的 λ
  → 所以小的 w 会被直接拉到 0

  sign 就是"告诉你这个数是正还是负"的函数，名字来自 signature（符号）。

                   ↑    ↑          ↑           ↑

  代码上的区别

  一句话：特征都有用选 Ridge，特征有垃圾选 Lasso。

> 异常点数据会造成权重系数过大、过小。尽量减少这些特征的影响（甚至删除某个特征

● 对，你这两句话总结得很准确。再理一下：

  第一句：异常点导致 w 极端

  正常数据：  [1, 2, 3, 4, 5]     → 学到 w = 2
  混入异常：  [1, 2, 3, 4, 500]   → 学到 w = 50
                            ↑
                       一个异常点把 w 拉飞了

  L2 Ridge：w 被压回合理范围 → w = 8（不完美但不离谱）
  L1 Lasso：如果这个特征本身就不靠谱 → w = 0（直接丢掉）

  第二句：高次项的 w 最容易爆

  y = w₁x + w₂x² + w₃x³ + w₄x⁴

  x=10 时各项的值：
    w₁ × 10     = 10w₁
    w₂ × 100    = 100w₂
    w₃ × 1000   = 1000w₃
    w₄ × 10000  = 10000w₄
                    ↑
            高次项放大效应巨大
            w₄ 稍微大一点，预测值就飞了

  正则化专门压制这些大 w
  → 高次项的 w 被压得最狠
  → 曲线变平滑，不会疯狂震荡

  不让 w 太极端，让模型更稳定

  具体体现在：
    ├── 异常点的影响被减弱（w 不会被少数极端样本带飞）
    ├── 高次项被压制（曲线平滑，不过拟合）
    └── 没用的特征被清理（L1 把垃圾特征归零）

> 有最后一个问题 我们不是全部标准化了 还有这种异常数据出现？                    

● 好问题。标准化和异常值是两个不同层面的事：

  标准化做了什么

  把每个特征缩放到均值=0、标准差=1

  原始：  [1000, 2000, 3000, 4000, 50000]
                                    ↑ 异常值

  标准化后：[-0.7, -0.5, -0.3, -0.1, 4.2]
                                      ↑ 还是异常值，只是数字变了

  标准化只改变了数字的刻度，没有改变数据之间的相对关系。 异常的还是异常。

  标准化解决的问题 vs 异常值的问题

  标准化解决：特征之间量级不同
    面积 1~5，收入 10000~50000
    → 标准化后都是 -2~2 范围
    → 梯度下降各方向步长均匀

  异常值的问题：某些样本本身就是错的或极端的
    房价 [30万, 35万, 28万, 32万, 500万]
                                   ↑ 这个不管怎么缩放都是离群的

  标准化管的：特征和特征之间的尺度差异
  异常值管的：样本和样本之间的偏离程度

  两个完全不同的问题
  标准化治不了异常值

  异常值哪来的

  1. 数据录入错误：房价 30万 打成 300万
  2. 真实存在的极端情况：大部分房子 30万，别墅 500万
  3. 传感器故障：温度突然跳到 999°

  这些在标准化之后还是会拉偏 w，所以正则化依然有用。

  标准化是给梯度下降创造好的训练条件，正则化是防止模型被极端数据带偏。
  各管各的，不冲突。
