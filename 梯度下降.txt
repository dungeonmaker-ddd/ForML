● 判断用正规方程还是梯度下降，核心就看 三个量：

  1. 特征数 n（最关键）

  - n < 10000：正规方程没问题，LinearRegression() 直接用
  - n > 10000：矩阵求逆 O(n³) 开始吃不消，换梯度下降
  - n > 100000（文本 TF-IDF、图像像素）：正规方程完全不可行，只能梯度下降

  这是最硬的分界线。因为瓶颈在 (XᵀX)⁻¹ 这个 n×n 矩阵的求逆。

  2. 样本数 m

  - 正规方程对 m 不太敏感（矩阵乘法是 O(m·n²)，m 大但 n 小照样快）
  - 梯度下降的 Batch 模式每步要扫全部 m 个样本，m 很大时单步慢——但可以用
  Mini-batch/SGD 绕过
  - m 极大 + 需要在线学习（数据持续流入）：只能梯度下降，因为正规方程每来一条新
  数据就得重算整个矩阵

  3. XᵀX 是否可逆

  - 特征之间有多重共线性（比如"面积"和"面积的平方"高度相关）→ XᵀX 接近奇异矩阵 →
   求逆数值不稳定或直接失败
  - 梯度下降不需要矩阵求逆，不受这个限制
  - sklearn 的 LinearRegression 内部用的是伪逆（SVD
  分解）缓解了这个问题，但本质上正规方程在共线性严重时结果不可靠

  实践判断流程

  特征数 n > 10000？
    → Yes → 梯度下降（SGDRegressor）
    → No  → 需要在线学习/增量学习？
              → Yes → 梯度下降（partial_fit）
              → No  → 正规方程（LinearRegression）就够了

  一句话：小特征用正规方程（精确、无超参数），大特征用梯度下降（可扩展、需调学习

> 梯度下降也是从导数开始的吗                                                    

● 是。梯度下降的每一步都在问同一个问题：loss 往哪个方向降最快，降多猛？

  回答这个问题的就是导数。

  导数告诉你两件事：

  1. 方向（正还是负）
  - 导数 > 0：函数在往上走 → w 应该减小
  - 导数 < 0：函数在往下走 → w 应该增大
  - 导数 = 0：到谷底了 → 停

  2. 幅度（绝对值大小）
  - 导数绝对值大 → 离谷底远 → 该走大步
  - 导数绝对值小 → 接近谷底 → 该走小步

  这就是为什么梯度下降有自动减速效应——越接近最优解，导数越小，步子自然变小，不容
  易冲过头。

  但你线性回归有两个参数 w 和 b，一个导数不够用。这时候就要偏导数：

  - ∂L/∂w：固定 b 不动，只看 loss 随 w 怎么变
  - ∂L/∂b：固定 w 不动，只看 loss 随 b 怎么变

  把两个偏导数合在一起，就是梯度：

  梯度 = [∂L/∂w, ∂L/∂b]

  一个向量，指向 loss
  上升最快的方向。取反方向，就是下降最快的方向——这就是"梯度下降"名字的来源。

  想看 MSE 的具体求导过程吗？还是先把导数的直觉吃透？

> 梯度下降的公司又是几个次方的呢                                                

● 梯度下降本身的复杂度是：

  O(m · n · k)

  - m = 样本数
  - n = 特征数
  - k = 迭代次数

  每一次迭代要扫一遍所有 m 个样本，每个样本有 n 个特征做一次乘法，重复 k
  次。三个数相乘，就这么简单。

  对比一下：

  ┌──────────┬──────────────────┬───────────────────┐
  │          │      复杂度      │ n=10000, m=100000 │
  ├──────────┼──────────────────┼───────────────────┤
  │ 正规方程 │ O(n³ + m·n²)     │ 10¹² + 10¹⁴       │
  ├──────────┼──────────────────┼───────────────────┤
  │ 梯度下降 │ O(m·n·k)，k=1000 │ 10⁹ × 10³ = 10¹²  │
  └──────────┴──────────────────┴───────────────────┘

  看起来差不多？关键区别在于 n 的次方：

  - 正规方程是 n³（立方）—— n 翻倍，计算量变 8 倍
  - 梯度下降是 n¹（线性）—— n 翻倍，计算量只变 2 倍

  所以特征少的时候正规方程快（一步到位不用迭代），特征一多梯度下降碾压（对 n
  只是线性增长）。

  而且 Mini-batch 还能进一步砍：每步不扫全部 m 个样本，只看 batch_size=64
  个，复杂度变成 O(64 · n · k)，跟 m
  无关了。这就是为什么它能处理几百万样本的数据。

> 那么梯度下降的计算过程是什么样的？  应该没有脱离人类计算的范畴吧              

● 完全没有，每一步都是你手算能做的加减乘除。

  拿最简单的例子：1 个特征、3 个样本、求 w 和 b。

  数据：
    x = [1, 2, 3]
    y = [2, 4, 5]

  初始化：w = 0, b = 0, 学习率 α = 0.1

  第 1 次迭代：

  第一步：算预测值 ŷ = w·x + b

    ŷ₁ = 0×1 + 0 = 0
    ŷ₂ = 0×2 + 0 = 0
    ŷ₃ = 0×3 + 0 = 0

  第二步：算误差 (ŷ - y)

    误差₁ = 0 - 2 = -2
    误差₂ = 0 - 4 = -4
    误差₃ = 0 - 5 = -5

  第三步：算梯度（偏导数）

    dw = 1/3 × (误差₁×x₁ + 误差₂×x₂ + 误差₃×x₃)
       = 1/3 × ((-2)×1 + (-4)×2 + (-5)×3)
       = 1/3 × (-2 + -8 + -15)
       = 1/3 × (-25)
       = -8.33

    db = 1/3 × (误差₁ + 误差₂ + 误差₃)
       = 1/3 × (-2 + -4 + -5)
       = 1/3 × (-11)
       = -3.67

  第四步：更新参数

    w = 0 - 0.1 × (-8.33) = 0 + 0.833 = 0.833
    b = 0 - 0.1 × (-3.67) = 0 + 0.367 = 0.367

  dw 是负数 → w 往正方向调（增大），合理，因为 y 都是正的，w=0 太小了。

  第 2 次迭代：用新的 w=0.833, b=0.367 再来一遍

  ŷ₁ = 0.833×1 + 0.367 = 1.20
  ŷ₂ = 0.833×2 + 0.367 = 2.03
  ŷ₃ = 0.833×3 + 0.367 = 2.87

  误差₁ = 1.20 - 2 = -0.80
  误差₂ = 2.03 - 4 = -1.97
  误差₃ = 2.87 - 5 = -2.13

  dw = 1/3 × ((-0.80)×1 + (-1.97)×2 + (-2.13)×3)
     = 1/3 × (-0.80 + -3.94 + -6.39)
     = 1/3 × (-11.13)
     = -3.71

  db = 1/3 × (-0.80 + -1.97 + -2.13)
     = -1.63

  w = 0.833 - 0.1 × (-3.71) = 0.833 + 0.371 = 1.204
  b = 0.367 - 0.1 × (-1.63) = 0.367 + 0.163 = 0.530

  看到没？误差变小了，梯度也变小了，步子自动变小了。

  迭代过程：

    w:   0 ──→ 0.833 ──→ 1.204 ──→ ... ──→ 1.5（真实值附近）
    b:   0 ──→ 0.367 ──→ 0.530 ──→ ... ──→ 0.33（真实值附近）

    loss: 15.0 ──→ 3.8 ──→ ... ──→ ≈0.17
           ↑                          ↑
         初始瞎猜                  接近最优

  就这些。没有高等数学，没有矩阵求逆，每一步就是：算预测 → 算误差 → 
  误差乘特征求平均 → 减一下。重复几百次就收敛了。

  计算机只是帮你重复这个过程几百上千次，速度快而已。

> 我们这里是利用误差来算偏导数吗 dw和db 就是梯度？直接累加就算更新梯度了吗      

● 对，拆开回答你三个问题：

  1. 是用误差算偏导数吗？

  是。MSE 损失函数对 w 求导，推到最后就是误差乘特征：

  L = 1/m × Σ(ŷᵢ - yᵢ)²

  对 w 求导（链式法则）：

  ∂L/∂w = 1/m × Σ 2(ŷᵢ - yᵢ) × xᵢ
                   ↑              ↑
                误差            特征值

  简化掉常数2（吸收进学习率）：

  ∂L/∂w = 1/m × Σ (ŷᵢ - yᵢ) × xᵢ    ← 这就是 dw

  ∂L/∂b = 1/m × Σ (ŷᵢ - yᵢ) × 1     ← 这就是 db
                                  ↑
                            b 的"特征"是1

  dw 和 db 的区别只在于乘的东西不同：dw 乘 xᵢ，db 乘 1。因为 b
  前面没有特征，相当于 b × 1。

  2. dw 和 db 就是梯度吗？

  dw 和 db 分别是偏导数。两个合在一起才叫梯度：

  梯度 = [dw, db]
       = [∂L/∂w, ∂L/∂b]

    一个参数 → 一个偏导数（导数）
    两个参数 → 两个偏导数组成的向量（梯度）
    100个参数 → 100个偏导数组成的向量（还是叫梯度）

  3. 直接累加就算出来了？

  对。回头看刚才的计算：

  dw = 1/m × (误差₁×x₁ + 误差₂×x₂ + 误差₃×x₃)
              \_________累加这三项__________/

  拆开看每个样本的贡献：

    样本1: 误差=-2, x=1  →  贡献 = -2 × 1 = -2
    样本2: 误差=-4, x=2  →  贡献 = -4 × 2 = -8
    样本3: 误差=-5, x=3  →  贡献 = -5 × 3 = -15
                                              ────
    累加 = -25，除以 m=3，dw = -8.33

  就是这么直接。每个样本贡献一个 "误差 × 特征值"，全部加起来取平均。

  注意一个细节——误差大且特征值大的样本，对梯度贡献最大。样本 3 贡献了
  -15，占总量的 60%，因为它误差最大（-5）而且 x
  也最大（3）。这意味着梯度下降会优先照顾"错得最狠"的样本。

  没有任何隐藏步骤，整个梯度下降就是：

  预测 → 算误差 → 误差乘特征 → 累加取平均 → 得到梯度 → 更新参数 → 重复

> 链式法则又是什么                                                              

● 链式法则就是求嵌套函数的导数的方法。

  你看 MSE 损失函数，它不是直接关于 w 的简单函数，而是一层套一层：

  最内层：  z = w·x + b          ← w 先算出预测值
  中间层：  e = z - y = ŷ - y    ← 预测值减真实值得到误差
  最外层：  L = e²               ← 误差平方得到损失

  套在一起：L = (w·x + b - y)²

  你要求 ∂L/∂w，但 L 不直接含 w，w 藏在最里面。

  链式法则说：一层一层往里剥，每层的导数乘起来。

  ∂L/∂w = ∂L/∂e × ∂e/∂z × ∂z/∂w

    ∂L/∂e = 2e           ← L=e² 对 e 求导
    ∂e/∂z = 1            ← e=z-y 对 z 求导，y是常数
    ∂z/∂w = x            ← z=w·x+b 对 w 求导，x是系数

  乘起来：

    ∂L/∂w = 2e × 1 × x
          = 2(ŷ - y) × x

  这就是刚才那个公式的来源。

  用个更直觉的例子。你踩油门，车速变，路程变：

  油门力度 → 车速 → 路程

  问：油门多踩一点，路程变多少？

  答：油门多踩1 → 车速多了3 → 路程每秒多了3
      但如果车速对路程的影响是×2
      那最终：1 × 3 × 2 = 6

  链式法则就是把每一环的"放大倍数"乘起来。

  回到梯度下降：

  w 变一点 → ŷ 变一点 → 误差变一点 → loss 变一点

  ∂L/∂w = 每一环的变化率相乘

        = 2(ŷ-y)  ×  x
          ↑           ↑

● 你问到点上了。先分清两个概念：

  求导：函数只有一个变量，直接求。

  f(x) = x²

  df/dx = 2x     ← 只有一个变量 x，没有歧义

  偏导：函数有多个变量，对其中一个求导，其他的当常数。

  f(w, b) = (w·x + b - y)²

  ∂f/∂w = ?     ← 把 b, x, y 全当常数，只看 w 的变化
  ∂f/∂b = ?     ← 把 w, x, y 全当常数，只看 b 的变化

  刚才链式法则那三步，每一步其实都是在做同一件事——对一个变量求导，其他都当常数：

  第一步：L = e²，对 e 求导

    e 是唯一变量 → 普通求导
    dL/de = 2e

    跟你学过的 x² → 2x 完全一样，只是把 x 换成了 e


  第二步：e = z - y，对 z 求导

    z 是变量，y 是常数（真实标签，固定的数）
    de/dz = 1

    跟 f(z) = z - 5 求导一样，常数项消失，系数是 1


  第三步：z = w·x + b，对 w 求导

    w 是变量，x 和 b 都当常数
    dz/dw = x

    跟 f(w) = 3w + 7 求导得 3 一样
    x 就是 w 前面的系数

  所以这三步没有任何新知识，每一步都是最基础的求导规则：

  规则1：xⁿ → n·xⁿ⁻¹      用在第一步：e² → 2e
  规则2：常数消失            用在第二步：z-y → 1
  规则3：系数保留            用在第三步：w·x → x

  链式法则只是说：你不能直接对 w 求导（因为 L 和 w 
  隔了好几层），但你可以每层分别求导，再乘起来。

  L 不认识 w
  但 L 认识 e，e 认识 z，z 认识 w

  所以问 L：e 变了你怎么变？  → 2e
  再问 e：z 变了你怎么变？    → 1
  再问 z：w 变了你怎么变？    → x

  串起来：w 变了，L 怎么变？  → 2e × 1 × x

  这就是全部原理。每一步都是初中学的求导，链式法则只是把它们串起来的方式。

  通用公式

  假设有 n 个特征，m 个样本：

  预测值：  ŷ = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

  用求和符号写：

            n
    ŷ =    Σ  wⱼ·xⱼ + b
           j=1

  梯度（对任意第 j 个参数）：

            1   m
    dwⱼ = ─── · Σ (ŷᵢ - yᵢ) · xᵢⱼ      j = 1, 2, ..., n
            m  i=1
                ↑          ↑
            第i个样本    第i个样本的
            的误差       第j个特征值

            1   m
    db  = ─── · Σ (ŷᵢ - yᵢ)              b 的"特征"永远是1
            m  i=1

  更新：

    wⱼ = wⱼ - α · dwⱼ      对每个 j = 1, 2, ..., n
    b  = b  - α · db

  就这三个公式，n=3 也好，n=10000 也好，完全一样。

  多项式特征呢？

  x² 或 x₁·x₂ 这种，在数学上只是造了一个新特征塞进去：

  原始特征：x₁, x₂

  加了多项式后：x₁, x₂, x₁², x₂², x₁·x₂
                                ↑     ↑      ↑
                            令x₃=x₁² x₄=x₂² x₅=x₁x₂

           5
    ŷ =   Σ  wⱼ·xⱼ + b
          j=1

                   1   m
    dwⱼ 还是  = ─── · Σ (ŷᵢ - yᵢ) · xᵢⱼ
                   m  i=1

  梯度下降根本不知道 x₃ 是 x₁ 的平方，它只看到一列数字。公式完全不变。

  矩阵写法（计算机实际跑的）

  当 n 很大时，不可能一个一个 wⱼ 写。用矩阵一行搞定：

  X: m×n 矩阵（m个样本，n个特征）
  w: n×1 向量（n个权重）
  y: m×1 向量（m个标签）

  正向：  ŷ = X · w + b              ← 一次矩阵乘法

  误差：  e = ŷ - y                  ← 一次向量减法

  梯度：  dw = 1/m · Xᵀ · e          ← 一次矩阵乘法
          db = 1/m · Σeᵢ            ← 一次求和

  更新：  w = w - α · dw
          b = b - α · db

  对应 numpy 代码：

  y_pred = X @ w + b           # 正向
  e = y_pred - y               # 误差
  dw = (1/m) * (X.T @ e)       # 梯度，Xᵀ乘误差
  db = (1/m) * np.sum(e)       # b的梯度
  w = w - lr * dw              # 更新
  b = b - lr * db

  不管 n=3 还是 n=100000，代码就这 6 行，一个字都不用改。X.T @ e 这一步 numpy
  内部用优化的矩阵运算，几百万维也跑得动。

随机梯度下降：随机选择一个样本的梯度值更新权重                                  
小批量梯度下降：随机选择小部分样本的梯度值更新权重                              
 这些是？                                                                       

● 就是你刚才学的那个公式里，求和的范围不同：

           1    ?
    dwⱼ = ─── · Σ (ŷᵢ - yᵢ) · xᵢⱼ
           ?   i=?
                ↑
             这里用多少个样本，就是唯一的区别

  全梯度下降（Batch GD）

  用全部 m 个样本：

           1    m
    dwⱼ = ─── · Σ (ŷᵢ - yᵢ) · xᵢⱼ
           m   i=1

  m=100000 时，每一步要算 100000 个误差再取平均

  优点：方向精确，直指谷底
  缺点：每步慢，m 大时内存吃不消

  随机梯度下降（SGD）

  随机挑 1 个样本（比如第 k 个）：

    dwⱼ = (ŷₖ - yₖ) · xₖⱼ

  不用求和，不用除以 m，就一个样本算一次

  优点：每步极快
  缺点：方向不准，锯齿形下降

  小批量梯度下降（Mini-batch GD）

  随机挑 B 个样本（B 通常 32 或 64）：

           1    B
    dwⱼ = ─── · Σ (ŷᵢ - yᵢ) · xᵢⱼ
           B   i=1

  优点：方向比 SGD 稳，速度比 Batch 快
  这是工业界默认选择

  用你熟悉的数字看效果。假设 m=90000 个样本，目标 w=5：

  Batch（每步用90000个）：

    步数少，每步方向准，但每步慢
    ──→──→──→──→──→ 到了 ✓
    5步到，但每步算90000个样本

  SGD（每步用1个）：

    步数多，方向抖，但每步极快
    ─→↗→↘→↗→↘→↗→→↘→→ 到了附近 ≈
    50步，但每步只算1个样本，总计算量反而少

  Mini-batch（每步用64个）：

    折中
    ──→─→──→─→──→ 到了 ✓
    10步，每步算64个，方向够稳，速度够快

  三种方法最终收敛到的结果几乎一样（MSE 是凸函数）。区别只是走的路径不同：

          Batch          SGD          Mini-batch

    w     ·              ·              ·
    ↑   ·              · ·  ·         ·  ·
    |  ·              ·  · ·  ·      ·    ·
    | ·             ·  ·    · ·     ·      ·
    |·             · ·       ·     ·        ·
    ★             ★              ★
    谷底          谷底附近        谷底

    直线走         锯齿走          小幅摇摆走
