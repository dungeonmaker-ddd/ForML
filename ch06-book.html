什么是拟合 & 模型复杂度

拟合 (Fitting)：用来表示模型对样本分布点的模拟情况。模型训练的本质就是让模型曲线尽可能贴合数据点的分布规律。

模型复杂度的不同，会直接导致三种截然不同的拟合结果：

低复杂度
y = kx + b
参数少，难以拟合规律
训练集表现极差
↓ 欠拟合
适中复杂度
y = ax² + bx + c
复杂度适中，正好拟合分布
训练集与测试集均好
→ 正好拟合
高复杂度
高次多项式
参数过多，训练集异常好
测试集表现很差
↑ 过拟合
Key
模型复杂度太低 → 欠拟合；复杂度太高 → 过拟合；复杂度适中 → 正好拟合。选择合适的模型复杂度是机器学习的核心挑战之一。

欠拟合 — Underfitting

现象：模型在训练集上表现很差，在测试集上表现也很差。

本质：模型学习到的特征过少，无法准确预测未知样本。模型连训练数据的规律都没学到。

原因
模型过于简单，参数太少，表达能力不足。
例如：用 y = kx + b（线性模型）去拟合明显的非线性分布数据，线性模型无法捕捉曲线规律。
对策
• 增加模型复杂度 — 使用更高阶的模型
• 增加特征 — 引入更多有效的输入特征
• 使用更强的模型 — 换用表达能力更强的算法
本质是提升模型的学习能力

过拟合 — Overfitting

现象：模型在训练集上表现很好，但在测试集上表现很差。

本质：模型学到的特征过多，包含了脏特征和噪声，相当于"钻牛角尖" — 记住了训练数据的个例而非真正规律。

原因
• 模型太复杂，参数过多（如高次多项式）
• 数据不纯，含噪声和异常值
• 训练数据太少，样本量不足以支撑模型复杂度
对策
• 正则化 — 约束参数，降低模型复杂度
• 异常值检测 — 清洗脏数据
• 特征降维 — 减少冗余特征
• 数据增强 — 扩大样本量
• 优化长尾数据 — 欠采样 / 过采样 / Focal Loss
本质是降低模型复杂度或提升数据质量

Loss 曲线诊断拟合状态

Loss（损失函数）：衡量模型预测值与真实值之间差异的度量。Loss 越小，模型越好。通过观察训练过程中 Loss 随 Step 的变化曲线，可以诊断拟合状态。

Loss 很大，下降很慢
学不到特征
后期验证集 Loss 反而上升
需要早停 (Early Stopping)
Train 和 Test 的 Loss
相近且收敛
诊断要点
• Train（蓝色实线）和 Test（红色虚线）都高 → 欠拟合
• Train 很低但 Test 后期反升 → 过拟合
• 两条线相近且都收敛到低值 → 正好拟合

泛化能力与奥卡姆剃刀

泛化 (Generalization)：模型在新数据集（非训练数据）上表现好坏的能力。即将具体的、个别的规律扩大为一般性能力。

泛化 = 模型在未见数据上的表现
泛化能力是衡量模型好坏的终极标准
欠拟合
学不到规律
泛化差
正好拟合
恰好学到规律
泛化好
过拟合
记住了噪声
泛化差
奥卡姆剃刀原则
给定两个具有相同泛化误差的模型，较简单的模型比较复杂的模型更可取。
简单模型更不容易过拟合，泛化能力往往更强。
核心思想
追求泛化能力 → 既不欠拟合也不过拟合 → 找到恰好的平衡点。
奥卡姆剃刀指导我们：同等效果下，优先选择更简单的模型。

章节小结

  • 拟合定义：拟合是模型对样本分布点的模拟情况，模型训练的本质就是让曲线贴合数据分布规律。
  • 欠拟合：训练集和测试集表现都差。原因：模型太简单。对策：增加复杂度、增加特征、换用更强模型。
  • 过拟合：训练集好但测试集差。原因：模型太复杂、数据不纯、样本不足。对策：正则化、异常值检测、特征降维、数据增强、优化长尾数据。
  • Loss 曲线诊断：通过观察 Train Loss 与 Test Loss 的走势，可判断欠拟合、过拟合或正好拟合。
  • 泛化是终极目标，奥卡姆剃刀指导模型选择 — 同等效果下优先选择更简单的模型。
一句话概括
模型不能太简单也不能太复杂，追求恰好拟合和良好泛化，这是机器学习模型选择的核心原则。
