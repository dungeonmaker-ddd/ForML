什么是拟合 & 模型复杂度

拟合 (Fitting)：用在机器学习领域，用来表示模型对样本点的拟合情况。训练的本质就是让模型曲线尽可能贴合数据分布的规律。

模型的复杂度不同，拟合效果截然不同。复杂度取决于模型参数数量和表达能力：

低复杂度
y = kx + b
参数少，难以拟合规律
→ 欠拟合
适中复杂度
y = ax² + bx + c
复杂度适中，正好拟合分布
→ 正好
高复杂度
高次多项式
参数过多，训练集异常好
→ 过拟合
Glossary
拟合 (Fitting)模型对样本点的模拟情况
模型复杂度 (Model Complexity)模型参数数量和表达能力

欠拟合

欠拟合 (Underfitting)：训练集表现很差，测试集表现也很差。模型学到的特征过少，无法准确预测未知样本。

原因
模型过于简单，参数少，表达能力不足。
例如：用 y = kx + b 去拟合非线性数据，直线无法捕捉曲线规律。
解决方案
• 增加模型复杂度
• 增加有效特征
• 换用更强的模型
本质：提升模型的学习能力
Glossary
欠拟合 (Underfitting)模型太简单无法捕捉数据规律

过拟合

过拟合 (Overfitting)：训练集表现很好，测试集表现很差。模型学到的特征过多，包含脏特征，钻牛角尖 —— 记住了噪声和个例，而非真正的规律。

原因
• 模型太复杂，高次方参数过多
• 数据不纯，含噪声和异常值
• 训练数据太少，脏特征被当成规律
解决方案
• 正则化 — 约束模型参数，防止过拟合
• 异常值检测 — 清洗脏数据
• 特征降维 — 减少冗余特征
• 数据增强 — 扩充训练数据量
• 长尾数据优化 — 欠采样 / 过采样 / Focal Loss
本质：降低模型复杂度或提升数据质量
Glossary
过拟合 (Overfitting)模型记住噪声而非真正规律
正则化 (Regularization)约束模型参数防止过拟合
数据增强 (Data Augmentation)扩充训练数据量的技术

Loss 曲线诊断

损失函数 (Loss)：衡量模型预测值与真实值之间的差异。Loss 越小，模型越好。通过观察训练过程中 Loss 的变化曲线，可以诊断拟合状态。

loss大且慢降
学不到特征
后期test反升
需要早停
双低且收敛
泛化良好
Glossary
损失函数 (Loss)衡量预测与真实值差异的度量
早停 (Early Stopping)在过拟合前停止训练

泛化能力与奥卡姆剃刀

泛化 (Generalization)：模型在新数据集（非训练数据）上表现好坏的能力。具体的、个别的扩大为一般的能力。

欠拟合
训练差，测试差
泛化差
正好拟合
训练好，测试好
泛化好
过拟合
训练好，测试差
泛化差

奥卡姆剃刀 (Occam's Razor)：给定两个具有相同泛化误差的模型，倾向选择较简单的模型。简单模型更不易过拟合，泛化能力往往更强。

核心原则
同等泛化效果，优先选简单模型。不欠不过，恰好平衡 — 这是模型选择的终极追求。
Glossary
泛化 (Generalization)模型对未见过数据的适应能力
奥卡姆剃刀 (Occam's Razor)同等效果优先选简单模型

章节小结

  • 拟合：用在机器学习领域，用来表示模型对样本点的拟合情况。模型复杂度决定拟合效果 — 过低欠拟合，过高过拟合，适中正好。
  • 欠拟合：训练集差、测试集也差。原因是模型过于简单，对策是增加复杂度、增加特征或换更强模型。
  • 过拟合：训练集好、测试集差。原因是模型太复杂或数据不纯，对策包括正则化、数据增强、特征降维等。
  • Loss 曲线：欠拟合双高且慢降，过拟合后期 test 反升出现 gap，正好拟合双低且收敛。
  • 泛化与奥卡姆剃刀：追求模型在新数据上的良好表现，同等泛化误差选较简单的模型。
模型不能太简单也不能太复杂，追求恰好拟合和良好泛化。
