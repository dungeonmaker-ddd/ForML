KNN vs 线性回归 — 两种学习范式

小弟
Claude的小弟
我之前学了 KNN 和手写数字，这和线性回归完全不同吗？完全不同嘛？
C
Claude
不是完全不同，都是监督学习，都从数据中学规律。但解决问题的方式截然不同 — 一个"记"，一个"算"。
小弟
Claude的小弟
不过手写数字感觉和 KNN 关系不大啊，主要就是完全计算大数组，然后对比差距值累加。
C
Claude
你说的就是 KNN。你觉得“和 KNN 关系不大”，是因为这个过程太直觉了 — 就是在比谁像谁，感觉没什么“算法”。但 KNN 的精髓就是这么朴素：不建模型，不学参数，纯靠比距离。
你的理解KNN 术语
大数组特征向量（784维）
对比差距值、累加计算欧氏距离
找最小的那个找最近的 K 个邻居
它是几我就猜几取邻居的标签作为预测
C
Claude
而线性回归是你要接触的第一个真正“学”出东西的模型。训练完留下一组参数 w 和 b，预测时代入公式 ŷ = wx + b，一步出结果。
KNN — 懒惰学习
存所有训练图片
新图来了 → 和每张算距离
找最近的 → 得答案
没有“学习”过程
线性回归 — 参数学习
用训练数据算出 w=2.5, b=30
新数据来了 → 代入公式
ŷ = 2.5x + 30 → 得答案
产出一组参数
Glossary
Lazy Learning 懒惰学习，不构建模型，预测时才计算
Regression 回归，输出连续数值（如房价、温度）

数据预处理与公式选择

小弟
Claude的小弟
那线性回归主要是对什么数据计算？也需要数据预处理吗？
C
Claude
和 KNN 手写数字不同，线性回归处理的不是图片像素，而是结构化的数值表格 — 面积、楼层、房龄这些数字列。预处理核心是特征缩放，让所有特征量级对齐。
小弟
Claude的小弟
所以会把所有特征都处理成 0 到 1 之间的数？
C
Claude
不是！缩放到 0~1 的叫归一化。线性回归常用的是标准化，结果大概落在 ±2 之间。
标准化 Z-score
x' = (x − μ) / σ
均值=0，标准差=1
95% 落在 ±2
线性回归常用
归一化 Min-Max
x' = (x − min) / (max − min)
映射到 [0, 1]
适合已知范围
如像素 0–255
小弟
Claude的小弟
我了解可能不止一种方程？有多种数学方程计算，那我们如何知道用哪种呢？是我们决定还是训练中决定？
C
Claude
人决定公式形状，机器算参数值。先用直线 ŷ=wx+b，效果不好就加平方项，还不行就换模型。机器学习里，人的核心工作不是算数，而是做选择。
Key — 易错点
标准化 ≠ 缩放到 0~1（那是归一化） | 标准化对象是特征 X，不是目标 y
“线性”的真正含义：对参数 w 是线性的，不是对数据 x。ŷ = w1×面积 + w2×面积² — x 有平方，但对 w 还是一次的 → MSE 还是碗形 → 还是线性回归。
Glossary
StandardScaler sklearn 标准化工具，均值=0 标准差=1
PolynomialFeatures 多项式特征展开，自动生成平方/交叉项

损失函数 — MSE 的碗形保证

小弟
Claude的小弟
所以我们刚才判断方程是否合适，使用的就是损失函数？
C
Claude
对，损失函数算出一个数字告诉你“有多差”。数字小 = 模型好，数字大 = 模型差。
指标公式特点用途
MAE1/n × Σ|y − ŷ|最直觉，单位和 y 一致评估
MSE1/n × Σ(y − ŷ)²放大大误差，处处可导训练
RMSE√MSE开根号，单位回到 y评估
小弟
Claude的小弟
那 MSE 的求导意义是什么？为什么要找最小的 w？
C
Claude
不是追求 w 本身小，是追求 MSE 最小时对应的那个 w — 那个 w 让预测最准。求导就是在问：从当前位置，往左走还是往右走能让 MSE 更小？
导数 < 0 → w 增大   导数 > 0 → w 减小   导数 = 0 → 最优解
导数告诉你方向和步幅 — 这就是梯度下降的全部原理
小弟
Claude的小弟
那怎么确定求导之后的曲线就是碗形呢？我还是不太好理解…
C
Claude
换个方式想。你在射箭 — 靶心是真实值 y，箭是预测值 ŷ。不管你往左偏还是往右偏，距离都变大。偏哪边都变大，只有正中间最小。这就是碗形 — 没有第二种方式让误差变成 0。
为什么碗形？
(y − wx)² 对 w 是二次函数
碗 + 碗 + 碗 = 还是碗
多参数 → 高维碗
永远只有一个最低点
为什么不可能两个坑？
假设 w=2 和 w=5 都让 MSE=0
意味着 2x = 5x
不可能（除非 x=0）
同一输入只有一个最准参数
Glossary
MSE Mean Squared Error，均方误差，训练时用
Convex 凸函数，碗形，只有一个最小值点

Day 03 · Page 4

正规方程与梯度下降

小弟
Claude的小弟
我们实际上是怎么求到那个最小值的呢？直接使用数学家们的公式？
C
Claude
对，直接用。两种方式，结果相同，按数据量选。
正规方程 — 一步到位
w = (XTX)−1XTy
代入数据，一步出所有 w 和 b
精确解，不用迭代
数据大时算不动
梯度下降 — 一步步逼近
从随机 w 开始
每次沿导数方向微调
重复几百上千次逼近最优
数据再大也能跑
需调学习率、迭代次数
小弟
Claude的小弟
可以给我讲解一下代码是怎么写的吗？不过这么说，这种方程的方式全都属于正规方程对吗？
C
Claude
不是，正规方程特指一个公式。三种写法，同一件事，结果完全一样：
手算版：自己算 sum_x, sum_y... 代入公式 矩阵版：np.linalg.inv(X.T @ X) @ X.T @ y sklearn：model.fit(X, y) ← 一行搞定 验证：面积 [50, 80, 100] 房价 [150, 250, 300] 随便猜 w=1, b=0 → MSE = 26300 (差得离谱) 正规方程 w=3.026, b=1.316 → MSE = 21.9 (很好)
小弟
Claude的小弟
我们学了导数但还没学矩阵，矩阵在线性回归里有什么用？这也算是计算机专属的写法？
C
Claude
矩阵就是“把数据打包，一次算完”。100万条数据要写100万行公式，有矩阵一行 y = X @ w + b 搞定。不用过度考虑细节，知道它在干什么就够了。
Iron Rule
正规方程和梯度下降不是递进关系，是同一步的两种工具。不是“正规方程不行才用梯度下降” — 它们算出的结果相同，只是计算方式不同。
Glossary
Normal Eq. 正规方程，直接求解析解的数学公式
Gradient Desc. 梯度下降，沿导数方向迭代逼近最优解

训练全流程 & 两个 fit

小弟
Claude的小弟 — 答题
拿到数据的步骤：预处理 → 做成矩阵 → 尝试一维 → 正规方程 → 效果不好再标准化 → 再不好就梯度下降。
C
Claude — 纠正
思路有，但顺序乱了。标准化是一开始就该做的前置步骤，不是“效果不好才做”。正规方程和梯度下降不是递进关系。正确顺序 ↓
1
数据预处理标准化 + 处理缺失值（前置步骤）
2
分割数据train_test_split → 80% 训练 / 20% 测试
3
选公式先用一次线性 ŷ = wx + b
4
训练model.fit() → 求最优 w 和 b
5
评估测试集上算 MSE / RMSE
6
效果不好→ 加平方项 → 还不行 → 换模型
小弟
Claude的小弟
这里有几个不好理解 — fit 是标准化数据吗？model.fit 就是训练吗？scaler.fit 计算的标准差后续在哪一步起作用？
C
Claude
两个 fit 同名不同义，这是初学者最大的坑：
scaler.fit
看一遍数据
记住均值和标准差
— 制作一把“尺子”
model.fit
训练模型
算出最优 w 和 b
— 内部跑正规方程
小弟
Claude的小弟
如果进行了预处理，预测时要怎么算？还是模型一开始就保存了预处理的参数？
C
Claude
scaler 保存了训练集的均值和标准差。新数据来了必须用同一把尺子标准化。训练集用 fit_transform（算尺子+标准化），测试集和新数据只用 transform（复用尺子）。绝对不能对测试集 fit_transform — 等于偷看答案。
scaler = StandardScaler() X_train = scaler.fit_transform(X_train) # 算尺子 + 标准化 X_test = scaler.transform(X_test) # 只标准化！ model = LinearRegression() model.fit(X_train, y_train) # 训练：算 w 和 b y_pred = model.predict(X_test) # 预测 mse = mean_squared_error(y_test, y_pred) # 评估
Glossary
fit_transform 算统计量 + 标准化（仅训练集）
transform 只标准化，复用已有统计量

Day 03 · Summary

章节小结

C
Claude
来，我帮你把今天学到的核心知识梳理一下：
  1. KNN “记”，线性回归 “算” — 第一个真正学出参数（w 和 b）的模型。
  2. “线性”指对 w 线性，不是对 x — x 有平方项，MSE 对 w 还是碗形。
  3. MSE 碗形是数学保证 — 平方 → 二次函数 → 唯一最低点 → 求导=0 即全局最优。
  4. 正规方程 & 梯度下降结果相同 — 不是递进关系，按数据量选。
  5. 两个 fit 同名不同义 — scaler.fit 记尺子，model.fit 算参数。测试集只用 transform。
Error Log — 易错清单
• 标准化 ≠ 缩放到 0~1（那是归一化，标准化是 ±2 正态分布）
• 标准化是前置步骤，不是“效果不好才做”
• 正规方程和梯度下降结果相同，按数据量选
• 多个 x → 预测一个 y，不是用少数特征预测其他全部特征
ŷ = w1x1 + w2x2 + … + b    MSE = 1/n × Σ(y − ŷ)²
w = (XTX)−1XTy（正规方程）  |  x' = (x − μ) / σ（标准化）
小弟
Claude的小弟
所以实际上，无论哪种类型都是在求那个最小值？然后验证求到的 w 是否能符合预测，不行就升维到平方或次方？
C
Claude
完全正确。这就是线性回归的全部逻辑。剩下的都是工具和细节 — 正规方程、梯度下降、标准化、正则化，全是围绕这个核心展开的技术手段。
