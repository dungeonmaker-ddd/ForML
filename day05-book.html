逻辑回归简介与 sigmoid 函数

小弟
Claude的小弟
Day 04 结尾提到逻辑回归。名字里有“回归”，它和线性回归是一回事吗？
C
Claude
名字有回归，但做的是分类。线性回归输出一个连续数值，逻辑回归输出一个类别。它专门解决二分类问题 — 答案只有两种：是/否、0/1。
应用场景类别 0类别 1
垃圾邮件检测正常邮件垃圾邮件
疾病诊断良性肿瘤恶性肿瘤
客户流失预测不流失流失
欺诈检测正常交易欺诈交易
小弟
Claude的小弟
线性回归输出的数值直接大于某个值就算 1 类，不行吗？
C
Claude
问题在于线性回归的输出范围是 (−∞, +∞)，没有上下限。分类需要一个 0 到 1 之间的概率值：0.9 表示 90% 是类别 1，0.2 表示 80% 是类别 0。我们需要一个“压缩器”把任意数值压到 (0,1) — 这就是 sigmoid 函数。
f(x) = 1 / (1 + e−x)
e ≈ 2.718（自然常数，固定数字）  输入任意实数 → 输出永远在 (0, 1) 之间
C
Claude
sigmoid 四个关键性质：
• 单调递增：x 越大，输出越接近 1
• 拐点在 x=0：f(0) = 0.5，正好是“五五开”的分界线
• 输出范围 (0,1)：永远不到 0 也不到 1，总留有余地
• 导数公式：f′(x) = f(x) × (1 − f(x))，用已有的预测值就能算导数，梯度下降时很高效
Key — 手算示例
sigmoid(2) = 1 / (1 + e−2) = 1 / (1 + 0.135) = 1 / 1.135 ≈ 0.88
→ 88% 的概率是类别 1。计算机会做这步，你只需记住 sigmoid 的 S 形曲线和意义。
Glossary
二分类 样本只分两类（0/1），逻辑回归的核心场景
sigmoid S 形激活函数，将实数映射到 (0,1) 概率区间
概率值 0~1 之间的数，表示样本属于某一类的可能性

数学基础 — 概率、极大似然、对数

小弟
Claude的小弟
逻辑回归的损失函数推导需要什么数学知识？
C
Claude
三个概念：联合概率、极大似然估计、对数函数。用一副扑克牌来说明前两个。一副标准牌 52 张，4 种花色。红色牌 26 张，K 有 4 张。
联合概率
两个独立事件同时发生
P(红 且 K) = P(红) × P(K)
= 1/2 × 1/13 = 1/26
概率相乘
条件概率
已知 B 发生，A 的概率
P(K | 红牌) = 2/26 = 1/13
P(红 | K) = 2/4 = 0.5
缩小了范围再计算
小弟
Claude的小弟
极大似然估计是什么？
C
Claude
核心思想：什么样的参数，最可能产生观测到的数据？举个例子 — 一枚不均匀硬币抛 6 次：正、反、反、正、正、正。正面概率记为 θ。每次独立，所以用联合概率：
L(θ) = θ4 × (1−θ)2
似然函数：产生观测数据的“可能性”  让 L(θ) 最大化 → θ = 4/6 ≈ 0.667
小弟
Claude的小弟
对数函数在这里起什么作用？
C
Claude
似然函数是概率连乘，几万个概率乘起来数值会小到计算机存不下。对数把乘法变加法：log(M×N) = log(M) + log(N)，还能把指数变乘法：log(Mn) = n×log(M)。取 log 后连乘变连加，数值稳定，且不改变极值位置（log 是单调递增的）。
Key — 为什么要学这三个概念
逻辑回归的损失函数推导路径：
联合概率（构建似然函数）→ 极大似然（最大化）→ 取对数（简化计算）→ 加负号 → 交叉熵损失
Glossary
联合概率 独立事件同时发生的概率 = 各自概率相乘
MLE 极大似然估计，让观测数据出现概率最大化
log 对数函数，乘法变加法的数学工具

逻辑回归原理 — 假设函数与交叉熵损失

小弟
Claude的小弟
逻辑回归的完整预测流程是怎样的？
C
Claude
三步：线性回归算分数 → sigmoid 变概率 → 阈值判类别。假设函数写成一行就是 h(w) = sigmoid(wTx + b)。线性回归的输出就是逻辑回归的输入。
1
线性计算z = w1×x1 + w2×x2 + b   输出一个实数（logits）
2
sigmoid 变概率p = 1/(1+e−z)   输出 0~1 之间的概率
3
阈值判断p > 0.5 → 类别 1   p ≤ 0.5 → 类别 0
小弟
Claude的小弟
logits 是什么？阈值 0.5 可以改吗？
C
Claude
logits 就是 wTx + b 的输出值，还没过 sigmoid 的原始分数，可以是任意实数。阈值可以调：医学场景降到 0.3（宁可误报不要漏掉患者），垃圾邮件提高到 0.7（宁可漏几封也不要误判正常邮件）。
小弟
Claude的小弟
损失函数为什么不能用 Day 03 学的 MSE？
C
Claude
sigmoid 是非线性的，MSE + sigmoid 会产生很多局部最小值，梯度下降会走到错误的谷底。所以用二分类交叉熵。推导用 Day 05 Page 2 学的数学：
1
单样本正确概率P = py × (1−p)(1−y)   y=1 时只剩 p，y=0 时只剩 1−p
2
n 个样本联合概率L = ∏ piyi (1−pi)(1−yi)   联合概率 = 各自相乘
3
取对数log L = ∑[ yi log(pi) + (1−yi) log(1−pi) ]   连乘变连加
4
加负号Loss = −∑[ y log(p) + (1−y) log(1−p) ]   最大化变最小化
Key — 优化方式
交叉熵损失没有解析解（不能用正规方程），只能用梯度下降迭代优化。
Day 04 提到过：“逻辑回归没有正规方程 — 这就是为什么要先学梯度下降。”
Glossary
logits 线性部分 wTx + b 的输出，未经 sigmoid 的原始分数
交叉熵 Binary Cross-Entropy，逻辑回归的标准损失函数
阈值 概率分界线，默认 0.5，可按业务场景调整

API 函数与癌症分类案例

小弟
Claude的小弟
逻辑回归的 API 和线性回归差别大吗？
C
Claude
接口几乎一样 — fit/predict/score 三件套不变。多了一个 predict_proba() 返回每个类别的概率值，这是分类模型特有的。
from sklearn.linear_model import LogisticRegression lr = LogisticRegression( solver='liblinear', # 求解引擎 penalty='l2', # 正则化类型 C=1.0 # 正则化力度（C 越小惩罚越重） ) lr.fit(X_train, y_train) # 训练 y_pred = lr.predict(X_test) # 预测类别 y_prob = lr.predict_proba(X_test) # 预测概率 [P(0类), P(1类)]
solver适用场景支持的正则化
liblinear小数据集L1, L2
sag / saga大数据集L2 / L1+L2
lbfgs中等数据集L2
小弟
Claude的小弟
solver 是什么？penalty 和 C 参数是什么意思？
C
Claude
solver 是用什么算法做梯度下降，不同算法适合不同数据量。penalty 是正则化类型，L1 可以让某些权重归零（自动选特征），L2 让所有权重变小。C 控制正则化力度 — 注意 sklearn 里 C 越大惩罚越轻，和教科书的 λ（越大惩罚越重）方向相反。
# ═══ 癌症分类完整流程（sklearn 内置数据集）═══ from sklearn.datasets import load_breast_cancer from sklearn.model_selection import train_test_split from sklearn.preprocessing import StandardScaler data = load_breast_cancer() # 569样本, 30特征 X_train, X_test, y_train, y_test = train_test_split( data.data, data.target, test_size=0.2, random_state=42) scaler = StandardScaler() X_train = scaler.fit_transform(X_train) # 训练集 fit+transform X_test = scaler.transform(X_test) # 测试集只 transform lr = LogisticRegression(solver='liblinear') lr.fit(X_train, y_train) print(f"准确率：{lr.score(X_test, y_test):.4f}") # ≈ 0.9737
Key — 流程和线性回归完全一致
加载数据 → 切分 → 标准化（梯度下降必须）→ fit 训练 → predict 预测 → score 评估
只是把 LinearRegression 换成了 LogisticRegression
Glossary
predict_proba 返回概率值而非类别，分类模型特有
C 参数 正则化力度，C 越小 → 惩罚越重 → w 越受限

分类评估 — 混淆矩阵、P/R/F1、ROC/AUC

小弟
Claude的小弟
准确率不够用吗？1000 个样本判对 990 个，准确率 99% 不是很高吗？
C
Claude
如果 1000 个样本里 990 个健康、10 个患病，模型全预测“健康”准确率也是 99% — 但一个患者都没找到。准确率在样本不均衡时会骗人，所以需要混淆矩阵做更细致的拆解。
预测：正例预测：反例记忆
真实：正例TP 真正例FN 伪反例漏掉了
真实：反例FP 伪正例TN 真反例误报了
C
Claude
记忆口诀：第一个字 T/F = 判对/判错，第二个字 P/N = 模型说的是正/反。基于这四个值计算三个核心指标：
精确率 P
TP / (TP + FP)
找出来的阳性，有多少是对的
关注：避免误报
场景：垃圾邮件
召回率 R
TP / (TP + FN)
真正的阳性，你找到了多少
关注：避免漏报
场景：癌症诊断
F1-score
2×P×R / (P+R)
精确率和召回率的调和平均
任一方低 → F1 被拉低
要求两者均衡时用
小弟
Claude的小弟
ROC 曲线和 AUC 又是什么？
C
Claude
ROC 曲线以 FPR（负样本被误判为正的比率）为 x 轴，TPR（正样本被正确找到的比率，就是召回率）为 y 轴。改变阈值会改变 FPR 和 TPR 的值，每个阈值画一个点，连起来就是 ROC 曲线。AUC = 曲线下面积，AUC=1 完美，AUC=0.5 等于随机猜，越大越好。曲线越靠近左上角 (0,1) 点，模型辨别能力越强。
from sklearn.metrics import classification_report, roc_auc_score # 完整分类报告（精确率/召回率/F1/support 一次输出） print(classification_report(y_test, y_pred, labels=[0,1], target_names=['良性','恶性'])) # AUC 值 auc = roc_auc_score(y_test, lr.predict_proba(X_test)[:,1]) print(f"AUC = {auc:.4f}")
Glossary
Precision 精确率，避免误报，TP/(TP+FP)
Recall 召回率，避免漏报，TP/(TP+FN)
AUC ROC 曲线下面积，0.5~1，越大分类能力越强

Day 05 · Summary

章节小结

C
Claude
今天从回归走到了分类，梳理核心知识：
  1. 逻辑回归 = 线性回归 + sigmoid — 把线性输出压缩为 (0,1) 概率值，解决二分类问题。
  2. sigmoid 函数 — f(x) = 1/(1+e−x)，S 形曲线，拐点 x=0 时 f=0.5，导数 f′=f(1−f)。
  3. 交叉熵损失 — Loss = −∑[y·log(p) + (1−y)·log(1−p)]，由极大似然法推导，只能用梯度下降优化。
  4. API — LogisticRegression(solver, penalty, C)，流程和线性回归一致，多了 predict_proba()。
  5. 混淆矩阵 — TP/FN/FP/TN 四个格子，第一个字 T/F = 对/错，第二个字 P/N = 模型说正/反。
  6. P/R/F1 — 精确率避免误报，召回率避免漏报，F1 是两者的调和平均。准确率在不均衡数据上会骗人。
  7. ROC/AUC — 不同阈值下 TPR vs FPR 的曲线。AUC 越大越好，0.5=随机，1=完美。
Error Log — 易错清单
• 逻辑回归名字有“回归”但做的是分类 — 因为它用了回归的思想来输出概率
• MSE 不能用于逻辑回归 — sigmoid 的非线性让 MSE 产生多个局部最小值
• sklearn 的 C 越大惩罚越轻 — 和教科书 λ（越大惩罚越重）方向相反
• 准确率高 ≠ 模型好 — 样本不均衡时全猜多数类也能准确率 99%
实战案例 — 电信客户流失预测
用逻辑回归预测客户是否流失（二分类），流程：缺失值处理 → 标签编码 → 特征选择 → 切分 → 标准化 → 训练 → classification_report + AUC 评估。 只用 3 个特征时 Recall ≈ 33%（漏掉 2/3 流失客户），需要加入更多特征或调低阈值来提高召回率。
