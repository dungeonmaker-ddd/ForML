KNN 算法思想与分类/回归流程

KNN 的核心思想极其直观：若一个样本在特征空间中的 K 个最近邻 大多属于某个类别，则该样本也属于这个类别。距离越近，样本越相似 —— 两轴差值是直角边，连线距离就是斜边（勾股定理）。

思路：为什么"近邻投票"有效？
数学事实：电影 A 打斗=100, 接吻=5；电影 B 打斗=95, 接吻=8。
d(A, B) = √((100−95)² + (5−8)²) = √(25+9) = 5.83
两者极近，因为特征值极相似。同类样本在特征空间中距离小 → 自然形成紧凑的簇。新样本到哪个簇的距离最小，就归属哪类。
本质：d(x, 同类中心) < d(x, 异类中心) → 投票等价于按距离划分决策区域。
分类流程
1. 计算距离
2. 升序排列
3. 取前 K 个
4. 多数表决
5. 归属最多类别
输出：离散标签
回归流程
1. 计算距离
2. 升序排列
3. 取前 K 个
4. 求均值
5. 均值作为预测值
输出：连续数值
Key
前 3 步完全相同，分歧在第 4 步：分类用投票、回归用均值。KNN 是惰性学习算法 —— 不构建模型，存储全部训练数据，预测时才计算。代价：预测时间 O(n×d)，内存占用大，高维时距离区分度下降（维度灾难）。
非参数模型 (KNN)
模型复杂度随数据量增长
· 数据即模型，不压缩为权重
· 1-NN 的决策边界是 Voronoi 图
· K 增大 → 边界平滑化
predict(x) = majority(y[k] for k in topK)
参数模型 (线性回归)
模型复杂度固定于参数量
· 学习 w1, w2, b 等有限权重
· 数据压缩为公式 y = wx + b
· 训练完成后可丢弃训练数据
predict(x) = w·x + b  (固定公式)
进阶：距离加权投票
普通投票中每个邻居权重相等。改进方案：wk = 1 / (distance(x, xk) + ε)，距离越近权重越大。
优点：更合理，利用了距离信息。缺点：计算更复杂。
场景：样本密度不均匀、数据分布不均时效果更好。sklearn 中设置 weights='distance' 即可启用。
加速：KD-Tree & Ball Tree
暴力搜索 O(n×d) 不可接受。KD-Tree 递归二分空间，预测时剪枝搜索，平均 O(d×log n)。Ball Tree 用超球体划分，适合高维。sklearn 默认 algorithm='auto'：样本少用 brute，特征少用 kd_tree，高维用 ball_tree。
Terms
KNNK-Nearest Neighbors，K 近邻算法
Lazy Learning惰性学习，不主动训练，预测时才计算距离
Non-parametric非参数模型，模型复杂度随数据增长，无固定参数量
Voronoi Diagram沃罗诺伊图，1-NN 的决策边界，空间按最近训练样本划分
KD-TreeK维树，递归二分空间加速最近邻搜索，O(d×log n)
Curse of Dimensionality维度灾难，高维空间中距离区分度急剧下降

K 值选择与 KNN API

K 值是 KNN 唯一核心超参数，不从数据中学来，需人为设定。K 值直接决定模型复杂度：

思路：K 值如何改变预测
训练集 y = [0, 0, 1, 1, 1]，其中第 3 个样本是噪声（标错了）。
· K = 1：新样本最近邻恰好是这个噪声点 → 预测为 1（错误）。训练误差 = 0，但一换数据集结果就变 → 高方差。
· K = 5：5 个邻居投票，噪声点只占 1 票，被 4 票淹没 → 预测正确。
· K = N：全体投票，类别 1 有 3 个，类别 0 有 2 个 → 永远预测 1，无论新样本在哪 → 高偏差。
数学事实：K=1 时训练集误差恒为 0（每个样本的最近邻是自己），但这不代表泛化能力强。
K 过小 → 过拟合
训练误差 = 0，测试可能仅 85%
· 对异常点高度敏感
· 坐标微变，分类就翻转
bias≈0, variance↑↑
K 过大 → 欠拟合
极端 K=N：永远预测多数类
· 受类别不均衡支配
· 丢失局部判别力
bias↑↑, variance≈0
实践
取较小奇数避免投票平局：3、5、7、9。最终用交叉验证遍历候选 K 值选最优。
偏差-方差视角 — K 是权衡的旋钮
K = 1：训练误差为 0（永远预测最近点的标签），但方差极高 — 换一批数据，预测可能完全不同。
K = N：偏差极高（永远预测多数类），但方差为 0 — 换任何数据结果都一样。
最优 K 在中间：bias² + variance 最小的平衡点。这是 K 值选择的数学本质。

sklearn API — 分类与回归只差一个类名：

API用途关键参数
KNeighborsClassifier 分类（投票法） n_neighbors=5
KNeighborsRegressor 回归（均值法） n_neighbors=5
Trap
predict 输入必须二维：predict([[1]]) 正确，predict([1]) 报错。sklearn 统一要求矩阵输入，即使只有一个样本也必须嵌套列表。评估方式：estimator.score() 或 accuracy_score()，两者结果一致。
最小可运行 KNN（4 行核心代码）
estimator = KNeighborsClassifier(n_neighbors=1)
X = [[0], [1], [2], [3]]   # 4 样本 1 特征，必须二维
y = [0, 0, 1, 1]   # 标签一维
estimator.fit(X, y)
estimator.predict([[4]])   # → [1]，必须二维输入
Terms
Hyperparameter超参数，训练前人为设定的参数（如 K 值）
Bias-Variance Tradeoff偏差-方差权衡，K 越小方差越高，K 越大偏差越高
Overfitting过拟合，模型学了噪声，训练好测试差
Underfitting欠拟合，模型太简单，训练测试都差
n_neighborsKNN API 中的 K 值参数，默认 5

距离度量

KNN 的核心在于计算距离。闵可夫斯基距离是统一框架，参数 p 不同就得到不同的距离：

概念思路：为什么需要多种距离？
取 A(1, 3)、B(4, 1)，差值 |Δx|=3, |Δy|=2：
· p=1 曼哈顿：3 + 2 = 5 — 维度贡献线性叠加，对异常维度不敏感
· p=2 欧氏：√(9+4) = 3.61 — 大差异被平方放大，更重视差异大的维度
· p→∞ 切比雪夫：max(3, 2) = 3 — 只看最大差异，其他维度完全忽略
过程思路：p 越大，最大差异的分量在求和中权重越高。p=1 平等对待所有维度，p=2 偏向大差异，p=∞ 极端到只看最差维度。选 p 就是在选对维度差异的敏感策略。
d = (∑|xi − yi|p) 1/p
闵可夫斯基距离 (Minkowski) — 不是新距离，是三种距离的统一表达
p 值距离名称公式直觉
p = 1 曼哈顿距离 ∑|xi − yi| 各维绝对值之和（L1 范数）
p = 2 欧氏距离 √∑(xi−yi)² 直线距离（L2 范数），默认
p → ∞ 切比雪夫距离 max|xi − yi| 只取最大分量（L∞ 范数）

手算示例：A(1,1)、B(2,2)、C(3,3)、D(4,4) 四点间欧氏距离：

点对计算过程欧氏曼哈顿切比雪夫
AB√((2−1)²+(2−1)²) = √21.41421
AC√((3−1)²+(3−1)²) = √82.82842
AD√((4−1)²+(4−1)²) = √184.24363
BC√((3−2)²+(3−2)²) = √21.41421
BD√((4−2)²+(4−2)²) = √82.82842
CD√((4−3)²+(4−3)²) = √21.41421
sklearn 参数
KNeighborsClassifier(metric='minkowski', p=2) — 默认欧氏。设 p=1 切换曼哈顿，p=3 试其他闵氏距离。algorithm 参数控制搜索方式：auto / kd_tree / ball_tree / brute。
p → ∞ 为什么等于切比雪夫？
当 p 趋于无穷大时，最大分量 |xi−yi| 的 p 次方远超其他分量之和。开 p 次根后，其他项贡献趋于 0，极限只剩 max|xi−yi|。
Key
量纲不同，距离就被大特征主导。年收入差 (10万)² = 1010，年龄差 (3岁)² = 9 — 收入贡献是年龄的 11 亿倍。所以 KNN 必须做特征预处理 → 下一页。
Terms
Euclidean欧氏距离，p=2，L2 范数，d=√∑(xi−yi)²
Manhattan曼哈顿距离，p=1，L1 范数，d=∑|xi−yi|
Chebyshev切比雪夫距离，p→∞，L∞ 范数，d=max|xi−yi|
Minkowski闵可夫斯基距离，统一以上三种距离的参数化框架
metric / psklearn 参数，metric='minkowski' + p=2 默认欧氏

特征预处理 — 归一化与标准化

KNN 基于距离计算，量纲差异会严重扭曲结果。预处理是 KNN 的前置刚需。

概念思路：为什么要预处理？
特征 A 范围 [0, 100000]（年收入），特征 B 范围 [20, 60]（年龄）。欧氏距离中 (100000−50000)² = 2.5×10&sup9;，而 (60−20)² = 1600。特征 A 的贡献是 B 的 156 万倍 — 年龄维度被完全淹没，KNN 实际上只按收入分类。标准化后两者方差 = 1，贡献均等。

过程思路：铁律的数学证据
# 错误做法：对测试集 fit_transform
scaler.fit_transform(X_test) → X_test 的 mean = 0, std = 1（已学到测试集统计量）
# 正确做法：只用 transform
scaler.transform(X_test) → X_test 的 mean ≠ 0, std ≠ 1（用训练集的参数变换）
错误做法中 mean=0 本身就是泄露的证据 — 测试集不可能恰好均值为 0，除非 scaler 已经"见过"了它。
MinMaxScaler
归一化
x' = (x − min) / (max − min)
映射到 [0, 1] 区间
弊端：max/min 若为异常值
→ 全部数据被过度压缩
鲁棒性差，仅适合精确小数据
StandardScaler
标准化
x' = (x − mean) / std
转换为 N(0, 1) 分布
优势：异常值对均值/std
影响可忽略，鲁棒性强
实践首选，适合嘈杂大数据
Key
归一化：x' = (x−min)/(max−min)，受极值支配，鲁棒差，适合小数据。标准化：x' = (x−mean)/std，受均值/std 驱动，鲁棒强，适合大数据。不确定时优先选标准化。
数学本质：为什么标准化有效？
标准化后每个特征方差 = 1 → 每个特征对距离公式的贡献相等。未标准化时，方差大的特征（如年收入）主导距离公式中对应的 (xi−yi)² 项。标准化的本质 = 对齐距离公式中每一项的量级。

实际数据对比 — 原始数据 [[90,2,10,40], [60,4,15,45], [75,3,13,46]]：

方法特征1 (90/60/75)特征2 (2/4/3)特性
归一化1.0 / 0.0 / 0.50.0 / 1.0 / 0.5受极值敏感
标准化1.22 / −1.22 / 0.0−1.22 / 1.22 / 0.0均值0 标准差1
正态分布 N(μ, σ) 与方差公式
方差 σ² = (1/n) ∑(xi − μ)²，衡量数据弥散程度。标准差 σ = √σ²。
标准正态：μ=0, σ=1。1σ 覆盖 68%，2σ 覆盖 95%，3σ 覆盖 99.7%。
1σ 到 2σ 之间：95% − 68% = 约 27%
Iron Rule — fit_transform vs transform
训练集：fit_transform — 学习均值/标准差 并 转换
测试集：transform — 只转换，复用训练集的统计量

为什么？测试集代表"未见数据"。如果对测试集 fit，模型间接"见过"测试集信息 → 评估虚高。这就是数据泄露 (Data Leakage)，面试能说出这个词 = 加分。
进阶：sklearn Pipeline 防泄露
数据泄露不只是 fit_transform 问题 —— 在全数据集上做特征选择再划分也是泄露。Pipeline 从根本上解决：
from sklearn.pipeline import Pipeline
pipe = Pipeline([('scaler', StandardScaler()), ('knn', KNeighborsClassifier())])
pipe.fit(X_train, y_train)   # scaler 只 fit 训练集
Terms
MinMaxScaler归一化，缩放到 [0, 1]，受极值敏感
StandardScaler标准化，转换为 N(0,1)，鲁棒性强
Data Leakage数据泄露，测试集信息渗入训练过程
fit_transform学习参数并转换（只用于训练集）
sklearn Pipeline串联预处理+模型，从根本上防止数据泄露

鸢尾花案例 — ML 完整 Pipeline

Iris 鸢尾花数据集：150 样本、4 特征（花萼/花瓣的长宽）、3 类别（setosa / versicolor / virginica）。用它走一遍 ML 通用 5 步流程 — 这个模板适用于所有 ML 项目：

过程思路：为什么必须这个顺序？（代码证明）
# 错误：先标准化，再划分
X_all = scaler.fit_transform(X_all)   # scaler 学了全部数据的 mean/std
X_train, X_test = train_test_split(X_all)
# 此时 X_test 已被全局 mean/std 污染 → 数据泄露

# 正确：先划分，再标准化

X_train = scaler.fit_transform(X_train) # 只学训练集
X_test = scaler.transform(X_test)      # 只变换 错误做法中，scaler 的 mean_ 包含了测试集样本的信息。模型间接"见过"测试集 → 评估准确率偏高 → 上线后性能下降。

.feature_names4 个字符串sepal length/width, petal length/width

Key
这 5 步就是 ML 项目的通用模板：获取 → 划分 → 预处理 → 训练 → 评估。无论换什么算法、什么数据集，骨架不变。后续手写数字识别也复用此流程。
完整代码模板（面试手写级）
# 1. 获取数据
from sklearn.datasets import load_iris
iris = load_iris()
# 2. 数据划分
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(
  iris.data, iris.target, test_size=0.2, random_state=22)
# 3. 标准化（铁律：fit_transform vs transform）
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)
# 4. 训练
from sklearn.neighbors import KNeighborsClassifier
knn = KNeighborsClassifier(n_neighbors=3)
knn.fit(X_train, y_train)
# 5. 评估
print(knn.score(X_test, y_test)) # → 准确率

交叉验证与网格搜索

为什么不直接用一次 train_test_split 评估？因为单次划分有运气成分 —— 同一模型 random_state=1 得分 0.93、random_state=2 得分 0.87，波动 ±0.06。N 次不同划分取平均后波动缩小到 ±0.02，评估更可信。

概念思路：为什么不直接最大化训练准确率？
K=1 时训练集准确率 = 100%（每个样本的最近邻是自己）。但测试集可能只有 85%。训练准确率高 ≠ 泛化好，这就是为什么需要独立评估。
单次 split 的问题：同一模型，random_state=1 时得分 0.93，random_state=2 时得分 0.87 — 波动 ±0.06。5-fold CV 取平均后波动缩小到 ±0.02，评估更可信。

过程思路：网格搜索为什么要穷举？
score(K) 不是 K 的单调函数 — 可能 K=3 得分 0.92，K=5 得分 0.96，K=7 又降到 0.91。没有公式能预测哪个 K 最优，只能遍历。
param_grid = {'n_neighbors': [1,3,5,7], 'p': [1,2]}
# 4 × 2 = 8 组超参 × 5 折 CV = 40 次训练
两个超参数形成 4×2 的网格，每个格点都要用 CV 评估一遍 — 这就是"网格"的含义。

网格搜索 (GridSearchCV) — 自动穷举超参数组合，每组都用 CV 评估：

参数/属性说明
estimator要优化的模型
param_grid超参候选值字典，如 {'n_neighbors': [1,3,5,7]}
cv交叉验证折数，常用 5 或 10
best_params_最优超参数组合
best_score_最优组合的 CV 平均分
best_estimator_最优模型（已自动 refit）
训练次数计算
总次数 = 超参组合数 × cv 折数
例：4 个 K 值 × 5 折 = 20 次训练
refit 陷阱
默认 refit=True：找到最优超参后自动用全部训练集重新训练。
best_estimator_ 已是最终可用模型。
GridSearchCV 完整代码
from sklearn.model_selection import GridSearchCV
# 构建网格
param_grid = {'n_neighbors': [1, 3, 5, 7, 9]}
gs = GridSearchCV(KNeighborsClassifier(), param_grid, cv=5)
gs.fit(X_train, y_train)
# 查看结果
print(gs.best_params_)   # {'n_neighbors': 5}
print(gs.best_score_)   # 0.9667
print(gs.best_estimator_) # 最优模型（已 refit）
# 导出全部结果

手写数字识别案例

MNIST 是机器学习分类的经典基准数据集（1999 年发布），包含 6 万训练 + 1 万测试 = 7 万张 28×28 灰度手写数字图像。

概念思路：为什么把图片展平成 784 维？
KNN 需要距离向量。每个像素是一个特征：pixel[100] 记录第 100 个位置的灰度值。数字"3"在特定像素有墨水（值 > 200），数字"1"在同一位置没有（值 ≈ 0）。展平 28×28 → 784 不丢信息，只改变数据形状。

过程思路：维度灾难的数学事实
d 维空间中，随机点间的距离分布有一个性质：
limd→∞ (dmax − dmin) / dmin → 0
即维度越高，最远点和最近点的距离差异趋于 0。d=4 时 dmax/dmin ≈ 2.5，d=784 时 ≈ 1.05。所有点几乎等距 → "最近邻"失去区分意义 → KNN 退化。这就是为什么 784 维需要 CNN 提取局部特征，而不是暴力比距离。
复用 Pipeline
同样的 5 步流程：加载 CSV → train_test_split → StandardScaler 标准化 784 维特征 → KNN + GridSearchCV 调优 K 值 → score 评估。证明了 Pipeline 的通用性。

章节小结

  • KNN 思想：K 个最近邻投票/均值做决策。非参数模型 —— 数据即模型，不压缩为权重。1-NN 决策边界是 Voronoi 图。惰性学习代价：O(n×d)，可用 KD-Tree 加速至 O(d×log n)。
  • K 值选择：K 过小 → 过拟合（高方差），K 过大 → 欠拟合（高偏差）。本质是 bias-variance tradeoff 的旋钮。取较小奇数，用 CV 选最优。
  • 距离度量：闵可夫斯基距离统一三种距离 — p=1 曼哈顿、p=2 欧氏、p→∞ 切比雪夫。sklearn 用 metric + p 参数控制。量纲差异会主导距离。
  • 特征预处理：归一化受极值支配（小数据），标准化受均值/std 驱动（大数据首选）。数学本质：标准化使每个特征对距离贡献相等。铁律：训练集 fit_transform，测试集只 transform — 防止数据泄露。Pipeline 从根本上解决。
  • ML Pipeline：获取 → 划分 → 预处理 → 训练 → 评估，5 步通用模板。鸢尾花和 MNIST 都复用同一骨架。两种评估方式：score() 和 accuracy_score()。
  • 模型调优：交叉验证消除划分随机性，网格搜索穷举超参组合。两阶段：CV 选最优 K → 全训练集 refit。训练次数 = 超参组合 × cv 折数。
  • MNIST 案例：7 万张 28×28 灰度图，784 维特征。KNN ~96% 但极慢，CNN 99%+。KNN 的价值：验证 Pipeline 通用性，体验维度灾难。
