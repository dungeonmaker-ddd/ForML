| # | 认知 | 实验文件 | 核心数据 |
|---|------|----------|----------|
| 1 | 计算机看到一维向量，不知道二维形状 | how_computer_sees | "0"vs"0"=23.7, "0"vs"1"=59.6 |

| 6 | KNN永远给答案，不会说"不知道" | garbage_input | 纯噪声→预测7(80%自信) |

| 9 | 少数像素贡献大部分距离 | distance_truth | 前5像素=58%总距离 |

## 🎯 核心公式

```
KNN预测一张图的计算量: N × D
  N = 训练样本数 (1437)
  D = 特征维度 (64 或 256 或 784)

欧氏距离: sqrt( (a0-b0)² + (a1-b1)² + ... + (a63-b63)² )
  每个像素都参与，但贡献不均匀

预测 = 找最近K个邻居 → 投票 → 多数赢
```

## 🔨 构建历程

### 学习路径
1. **从直觉出发**: 图片=像素矩阵=特征向量 → KNN算距离投票
2. **发现问题**: 位置敏感、噪声脆弱、8/6/9混淆
3. **理解预处理**: 二值化→居中→归一化，在二维阶段完成
4. **深入距离**: 少数像素主导距离，噪声让排名翻转
5. **理解K值**: 投票人数，抗噪声的核心机制
6. **理解局限**: 手工特征丢信息，KNN计算量=N×D

### 关键发现
- 标准化不是万能的，图像数据可能不需要
- 放大≠信息增加，插值只是"涂抹"
- KNN不训练，所有计算推迟到预测时
